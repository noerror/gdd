# 게임 캐릭터 컨셉 설계 가이드

1. **개요**: 캐릭터 컨셉 설계의 중요성과 디자이너-아티스트 간의 명확한 역할 분담 및 협업의 필요성
2. **캐릭터 컨셉 기획**: 기본 설정부터 개발 방법론, 캐릭터 유형별(PC, NPC, 몬스터) 상세 설계 전략과 비주얼 지시 작성법
3. **디자이너-아티스트 협업 가이드**: 작업 프로세스 단계별 산출물, 효과적인 피드백 방법과 중간 점검 체크리스트
4. **결론**: 고품질 결과물 도출을 위한 직군별 핵심 원칙과 성공적인 협업 마인드셋
5. **부록. 아티스트 작업 팁**: 색상, 실루엣, 디테일 등 시각적 완성도를 높이기 위한 아티스트 전용 실무 가이드와 검증 항목

# 1. 개요

**캐릭터 컨셉 설계의 중요성**

- 설정과 시각 표현의 일치가 게임 완성도에 미치는 영향
- 컨셉 붕괴 사례와 비용 손실
- 효과적인 협업이 품질과 효율에 미치는 영향

**디자이너-아티스트 역할 구분**

| 역할 | 디자이너 영역 | 아티스트 영역 |
|------|-------------|-------------|
| 결정 사항 | 나이, 성격, 직업, 역할 | 정확한 비례, 표정 연출 |
| 방향 제시 | 분위기 키워드, 컨셉 | 세부 디테일, 창의적 해석 |
| 산출물 | 기획서, 레퍼런스 | 완성 아트워크 |

# 2. 캐릭터 컨셉 기획

## 2-1. 기본 설정 요소

### **메커니즘**
- 이름, 성별, **나이 (설정 나이 = 외형 나이 필수 일치)**
- 직업/역할, 소속, 배경 스토리

### **특징 및 차별점**
- **컨셉 일관성**: 27살 설정인데 10대 외형 = 컨셉 붕괴
- **불일치 예외**: 스타일적 선택(카툰/SD)이나 의도적 디자인일 경우 **의도/근거/목표 연령**을 문서에 명시
- **시각적 아이덴티티**: 한눈에 역할/성격 파악 가능
- **성격 반영**: 소품, 복장, 표정으로 성격 시각화

### **설계 고려사항**
- 설정 나이와 체형/얼굴/주름 일치 여부
- 성격이 외형에 드러나는 요소 (대사, 행동, 아이템)
- 세계관 내 문화적 적합성

### **디자이너 작성 체크리스트**
- [ ] 나이-외형 일치 명시
- [ ] 핵심 성격 3가지 정의
- [ ] 역할을 나타내는 필수 아이템 지정
- [ ] 한 문장 배경 스토리 작성

### 비주얼 일관성과 캐릭터 스케일링

**세계관 내 시각적 통일성**
- 모든 캐릭터가 같은 게임 세계에서 공존하는 것처럼 설계
- 현실적 인간과 카툰 캐릭터의 혼합 피하기
- 색상 채도 조정으로 시각적 균형 맞추기

**프로포션 표준화**
- 카메라/장르 기준 가독성 확보 (시야 거리에서 실루엣·표정·아이템 구분 가능)
- 원작과 다른 크기라도 게임 내 균형 고려
- 디테일 레벨 평준화 (단순 캐릭터에 디테일 추가, 복잡한 캐릭터는 간소화)

```
[예시 기준]
- 탑다운 게임: 1-3미터 범위 (Super Smash Bros. 스타일)
- 3인칭 액션: 1.5-3.5미터 범위 (실제 인간 크기 비율)
- 횡스크롤 플랫폼: 0.6-1.8미터 범위 (화면 크기에 맞춘 축소)
```

## 2-2. 캐릭터 선정과 테마 개발

### 캐릭터 선택 기준
게임 내 캐릭터를 선정할 때는 다음과 같은 요소들을 고려해야 합니다:

**인기도와 대표성**
- 게임 역사적 중요성 (특정 시스템이나 시대 대표)

**다양성과 균형**
- 캐릭터 유형의 균등 분배
- 과도한 특정 프랜차이즈 집중 피하기
- 새로운 캐릭터와 기존 캐릭터의 조화

**기술적/비즈니스적 고려사항**
- 기존 에셋 재활용 가능성 (모델, 애니메이션)
- 개발 리소스와 일정 고려

### 테마 개발 방법론
각 캐릭터는 **단 한 문장으로 요약 가능한 고유한 테마**를 가져야 합니다:

**테마 예시**
```
- "무거운 화력 전문가" (Heavy Weapons Specialist - Snake)
- "변신 능력을 가진 펫 파이터" (Puppet Fighter - Rosalina & Luma)
- "제한 사용 공격의 마법사 전사" (Sword and Magic, Limited-Use Attacks - Robin)
```

**유니크함 확보 원칙**
- 기존 캐릭터와의 차별화된 능력이나 특성 필수
- 게임 내에서만 가능한 독특한 메커닉 활용
- 예측 가능한 캐릭터보다 색다른 컨셉 선호

## 2-3. 컨셉 개발 방법론

### 마인드맵 확장법

**아이데이션 방법**: 기본 단어 → 연관 단어 → 독창적 조합
```
악마 → 착한악마 → 천사 → 타락천사
도적 → 의적 → 로빈훗 → 정의로운 도적
```

**특징**: 차별화된 컨셉 발굴, 예측 가능성 회피

**설계 고려사항**: 3-4단계 확장, 상반된 개념 조합

### 역사적/문화적 모티브 활용

**아이데이션 방법**: 실제 인물의 대중적 이미지 + 픽션 변형

**사례**:

| 역사적 인물 | 대중 이미지 추출 | 판타지 변환 |
|-----------|---------------|-----------|
| 토마스 에디슨 | 발명가, 흰머리, 19C 복장 | 앤티크 복장 마법사 |
| 잔 다르크 | 성녀 전사 | 신성 기사 |
| 장영실 | 과학자 | 미래지향적 기술자 |

**설계 고려사항**: 친숙함 + 독창성, 실제 이미지는 레퍼런스로 제공

### 심리학 기반 컨셉 설정

**아이데이션 방법**: 내면 상태를 비주얼 요소로 변환

**적용 예시**:
- 야망 있는 캐릭터 → 높은 곳에 위치한 거처
- 개방적 성격 → 창문 많은 디자인, 밝은 색상
- 불안정한 상태 → 떠있는 형태, 지지대 없음
- 방어적 성격 → 폐쇄적 복장, 갑옷 강조

**설계 고려사항**: 지시서에 심리적 배경 명시

## 2-4. 캐릭터 유형별 컨셉 설계

### 플레이어 캐릭터 (PC)

**게임플레이 메커닉**
- 다각도 표현 필수: 정면/측면/반측면/하이앵글/로우앵글
- 7-8등신 비례 (균형잡힌 비율)
- 모델링 시트: 정면/측면/후면/상세도

**특징 및 차별점**
- 3D 연동 가능성 고려한 설계
- 직업별 명확한 시각적 차별화
- 코스튬 베리에이션으로 확장성 증명

**설계 고려사항**
- 각도별 무너지지 않는 구조 요청
- 표정 레퍼토리 (기본/전투/승리 등)
- 직업별 필수 아이템과 체형

**직업별 차별화 포인트**

| 직업 | 체형 | 필수 아이템 | 색상 경향 |
|-----|------|-----------|----------|
| 궁수 | 슬림, 민첩 | 활, 화살통 | 자연색 (녹색, 갈색) |
| 마법사 | 평균, 우아 | 지팡이, 로브 | 신비색 (보라, 청색) |
| 기사 | 근육질, 넓은 어깨 | 검, 방패, 갑옷 | 금속색 (은, 금) |
| 힐러 | 부드러운 곡선 | 성물, 약초 | 따뜻한 색 (흰색, 황금) |

**작성 예시**
```
[PC - 전사형]
- 비례: 8등신, 넓은 어깨 강조
- 필수 각도: 정면, 3/4, 로우앵글 (위압감)
- 표정: 강한 눈빛, 단호한 입매, 긴장된 광대뼈
- 복장: 실용적 갑옷, 전투 흔적
- 금지: 과도한 장식, 귀여운 느낌
```

### NPC 캐릭터

**게임플레이 메커닉**
- 6-7등신 비례 (친근한 느낌)
- 역할별 시각적 코드 명확화
- 반복 사용 가능한 라이브러리화

**특징 및 차별점**
- 성격-외모 일치 우선순위
- 아이템으로 스토리텔링 강화
- 주인공과의 시각적 구분 (채도 낮춤)

**설계 고려사항**
- 노인 캐릭터 포함으로 연령 스펙트럼 확보
- 대사/행동과 일치하는 외모
- 상점주인/퀘스트NPC/배경인물 구분

**역할별 디자인 포인트**
```
상점 주인: 풍요로운 체형, 상품 관련 소품
퀘스트 NPC: 모험가 느낌, 지도/아이템
배경 인물: 단순화된 디테일, 낮은 채도
```

### 몬스터/크리쳐

**게임플레이 메커닉**
- 인체 + 동물 조화 (생물학적 타당성)
- 행동 패턴 기반 디자인 (빨판 → 벽 붙기)
- 앞/옆/뒤 뷰 + 질감 참고 제공

**특징 및 차별점**
- 설득력 있는 비인간 형태
- 생동감 있는 색상 대비
- 공격 패턴과 형태의 논리적 연결

**설계 고려사항**
- 동작 자연스러움 (문어 다리 휘어짐 등)
- 기존 미디어 성공 사례 참고 (아쿠아맨 해파리)
- 빨판/껍질 등 생물학적 세부 명시

**디자인 지시 예시**
```
[해파리형 크리쳐]
- 형태: 투명한 상체 + 촉수 다리
- 행동: 물 속 유영, 전기 공격
- 색상: 청록색 본체 + 빛나는 촉수
- 참고: 아쿠아맨 해파리 장면
```

## 2-5. 세계관 연계 설정

### 캐릭터 코어
- 진영, 관계도, 라이벌 구축
- 주인공과 대조되는 설계

### 라이벌 캐릭터 구축 원칙

| 요소 | 주인공 | 라이벌 (대비) |
|------|--------|-------------|
| 성격 | 정의로움 | 냉소적 |
| 전투 | 빠른 공격 | 강력한 일격 |
| 색상 | 파랑/흰색 | 빨강/검정 |
| 체형 | 슬림 | 근육질 |

### 문화 코드 반영
- 판타지 세계관 → SF 요소 (LED, 금속 소재)
- 현대 세계관 → 현실성 강조
- 한국/서양/일본 요소 적절 혼합

**설계 고려사항**
- 대비 요소 3가지 이상 명시
- 색상 팔레트를 대조적으로 설정
- 세계관 내 논리적 타당성 확보

## 2-6. 비주얼 지시 작성법

### 분위기 키워드 설정
```
[필수 작성 형식]
- 전체 분위기: (예: 카리스마 있는 여전사)
- 체형: (날씬/근육질/통통/평범)
- 복장 스타일: (중세 갑옷/현대복/SF 슈트/판타지 로브)
- 주요 색상 3-4가지: 메인/서브/포인트
- 필수 아이템: (구체적으로)
```

### 레퍼런스 제공 규칙
- 유사 캐릭터 이미지 2-3개 필수
- "○○ 게임의 △△ 느낌" 구체적 예시
- 복장/소품 참고 이미지 별도 제공
- 캐릭터, 환경, UI 등 중요 요소별 참고 자료 준비
- 색상 팔레트, 스케일, 타이포그래피 등 스타일 요소 연구

### 컨셉 콜라주 활용
- 수집된 레퍼런스 이미지를 콜라주 형태로 정리
- 팀 내 공개 게시로 피드백 수집 및 팀 정렬 유도
- 게임 비전을 공유하고 기대감 조성

### 금지 사항 명시
```
[금지 항목 예시 - 기준 목적 명시]
- (등급/정책) 과도한 노출 ❌
- (판독성) 지나치게 화려한 장식 ❌
- (세계관 톤) 귀여운 느낌 ❌
- (판독성) 너무 디테일하게 파지 말 것 (전체 밸런스 우선)
```

```

프로젝트: [게임명]
캐릭터: [캐릭터명]
제작일: [YYYY-MM-DD]
담당자: [이름]

2. 게임플레이 역할
   - 역할: [탱커/딜러/서포터/힐러 등]

3. 시각적 실루엣 (3개 키워드)
   - [예: 삼각형, 긴 로브, 수정 구슬]

4. 재질/소재 키워드 (3개)
```

## 3-2. 효과적인 피드백 방법

### 나쁜 피드백 vs 좋은 피드백

**❌ 나쁜 피드백 예시**
```
- "별로예요"
- "더 예쁘게 해주세요"
- "느낌이 안 나요"
- "멋있게 그려주세요"
```

**✅ 좋은 피드백 예시**
```
- "27살 설정인데 10대로 보입니다. 턱선을 각지게, 눈가에 주름 추가해주세요"
- "전사 역할인데 약해 보입니다. 어깨 넓이 1.2배, 팔 근육 강조해주세요"
- "개방적 성격인데 폐쇄적으로 보입니다. 표정을 미소로, 의상 색상을 밝게 조정해주세요"
```

### 피드백 작성 원칙
1. **문제점 명확화**: "무엇이" 문제인지
2. **근거 제시**: "왜" 문제인지 (설정 기반)
3. **구체적 해결책**: "어떻게" 수정할지
4. **우선순위 표시**: 필수/선택 구분

# 4. 결론

**디자이너 핵심 원칙**
1. **명확한 설정**: 나이=외형, 성격=비주얼
2. **구체적 지시**: 키워드 5개 이상, 레퍼런스 필수
3. **창의성 여지**: 40% 자유도 확보
4. **건설적 피드백**: 근거+해결책 제시

**아티스트 핵심 원칙**
1. **입력 검증**: 모호한 지시 = 수정 지옥
2. **체계적 작업**: 큰 형태 → 세부 디테일
3. **품질 체크**: 제출 전 10초 체크 필수
4. **소통**: 불명확한 부분 즉시 질문

**협업 핵심 원칙**
1. **역할 분담**: 디자이너 + 아티스트
2. **단계별 검토**: 러프→채색→최종 3단계
3. **문서화**: 모든 결정 기록
4. **상호 존중**: 전문성 인정
