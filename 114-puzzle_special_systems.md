# 퍼즐 및 특수 플레이 시스템 작성 가이드

## 1. 목적 (Purpose)
퍼즐 및 특수 플레이 시스템을 일관된 품질로 설계하고, 난이도 조절과 플레이어 재미를 보장하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 퍼즐 메커닉, 미니게임, 특수 모드(타임어택, 로그라이크 등), 난이도 밸런싱, 플레이어 피드백 시스템
- **제외**: 3D 모델링 세부 구현, 사운드 디자인 구체적 구현, 일반 게임플레이 메커닉(이동/전투 등)

## 3. 설계 목표 (Design Goals)

1. **접근성과 재미 (Accessible Fun)**: 모든 난이도에서 플레이어가 성취감을 느낄 수 있도록 퍼즐 해결의 즐거움을 최우선으로 고려합니다.
2. **균형 잡힌 도전 (Balanced Challenge)**: 좌절과 성취의 적절한 밸런스를 유지하며, 점진적 난이도 커브를 제공합니다.
3. **시스템 일관성 (Systemic Coherence)**: 게임 내 다른 시스템과 조화를 이루며, 전체 게임플레이 흐름에 자연스럽게 통합됩니다.
4. **혁신적 메커닉 (Innovative Mechanics)**: AI 통합, 컨텐츠 우선 접근 등 새로운 게임플레이 가능성을 탐색합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 시스템 개요 (Overview)
- **퍼즐/특수 시스템 개요**: 목적, 목표, 기본 메커닉
- **코어 질문**: 제공하는 재미, 난이도 조절 방식, 게임 흐름에서의 역할, 성취감·좌절감 밸런스

### 4.2 퍼즐 설계 (Puzzle Design)
- **퍼즐/챌린지 구성**: 각 퍼즐의 구조, 해결 방법, 난이도
- **난이도 밸런싱**: 난이도 레벨, 조정 메커닉, 플레이어 피드백
- **검수 기준**: 퍼즐별 캐치(모순)·가정(함정)·계시(발견) 3요소와 정량 채점표(해법 수, 필요 수순, 순간별 선택지 수, 신규 개념 수)

### 4.3 특수 모드 & 통합 (Special Modes & Integration)
- **특수 모드**: 타임어택, 로그라이크 등 변형 모드
- **통합 방식**: 메인 게임플레이와의 연결점
- **접근성 옵션**: 난이도 조정, 힌트 시스템 (선택 사항)

### 4.4 선택 및 심화 (Optional & Advanced)
- **힌트 시스템**: 시간 경과에 따른 단계적 힌트 제공, 광고 시청 후 해답 공개(모바일)
- **접근성**: 퍼즐 스킵 기능, 패턴의 색상/모양 이중 코딩(Color Blindness)
- **절차적 생성**: 퍼즐 패턴의 알고리즘 생성, 난이도에 따른 동적 배치

## 5. 관련 개념 (Key Concepts)

### 퍼즐 메커닉
- **Puzzle Loop**: 플레이어 입력 → 시스템 응답 → 결과 확인 → 다음 액션의 반복 사이클
- **Difficulty Curve**: 퍼즐의 복잡도가 점진적으로 증가하는 구조
- **Feedback System**: 퍼즐 진행 상태를 플레이어에게 전달하는 메커닉

### 특수 시스템
- **Special Mode**: 기본 게임플레이를 변형하는 추가 규칙이나 제약
- **Meta Puzzle**: 게임 세계관과 연결된 스토리텔링적 퍼즐

### 퍼즐 문법
- **캐치·가정·계시**: 좋은 퍼즐의 3요소 — 논리적 충돌(캐치), 당연해 보이는 오답으로의 유도(가정), 규칙의 비자명한 귀결 발견(계시)
- **사인포스팅 (Signposting)**: 대사·오브젝트에 단서를 흩뿌려 사고 재료를 제공하는 기법. 해법 노출과 구분됨
- **지식 게이트 (Knowledge Gate)**: 능력·아이템이 아니라 '알아낸 사실' 자체가 진행을 여는 구조 (Outer Wilds, Return of the Obra Dinn)

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 퍼즐 설계
- [ ] 퍼즐 목표가 명확하게 정의되어 있으며, 논리적 해결책을 가지는가?
- [ ] 난이도 레벨이 점진적으로 구성되어 있는가?
- [ ] 플레이어 피드백(힌트, 진행 표시)이 충분하며, 접근성 옵션이 제공되는가?
- [ ] 각 퍼즐이 캐치·가정·계시 3요소를 갖추었는가?
- [ ] 정답을 깨달으면 실행은 수초 내에 끝나는가? (실행이 어려우면 퍼즐이 아니라 손재주 시험)
- [ ] 오답에 제네릭 응답("그건 안 돼") 대신 왜 안 되는지와 실마리를 주는가?
- [ ] 힌트는 충분한 지연 시간 후 제공되며, 빈도 설정·비활성 옵션이 있는가?

### 6.2 시스템 통합
- [ ] 특수 모드가 기본 게임플레이를 보완하며, 게임의 전체 밸런스에 영향을 미치지 않는가?
- [ ] 시스템이 게임의 독특한 정체성을 강화하는가?
- [ ] 컨텐츠 우선 접근법으로 상상한 장면이 효과적으로 구현되었는가?

### 6.3 혁신 & 완성도
- [ ] AI 훈련 메커닉이 교육적 가치를 제공하면서 게임적 재미를 유지하는가?
- [ ] 퍼즐 시스템이 플레이어의 학습 과정을 시각적으로 피드백하는가?
- [ ] 기술적으로 구현 가능한 난이도이며, 성공률이 수치로 확인 가능한가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 점진적 난이도**
- **좋은 예 (Good)**: "기본 퍼즐: 3x3 그리드 패턴 매칭 → 중급: 시간 제한 + 복잡한 패턴 → 고급: 다중 레이어 + 역방향 해결 요구"
- **안티패턴 (Bad)**: "플레이어가 50개의 변수를 동시에 추적해야 하는 퍼즐" (구현 불가능한 복잡도)

**Case 2: AI 기반 퍼즐 (Little Learning Machines)**
- **좋은 예 (Good)**: "플레이어가 AI 캐릭터를 실시간으로 훈련시켜 퍼즐 해결: 샘플 문제 생성 → AI 학습 파라미터 조정 → 결과 관찰 및 개선, 학습 진행률 실시간 표시"
- **안티패턴 (Bad)**: "AI 메커닉이 퍼즐의 본질적 재미를 저해하는 기술적 과시"

**Case 3: 컨텐츠 우선 디자인 (Salt and Sanctuary)**
- **좋은 예 (Good)**: "상상한 장면(공중에 매달린 상태에서 콤보 공격)을 먼저 구현하고 주변 메커닉을 반복적으로 조정하여 정교한 히트박스와 콤보 시스템 구축"
- **안티패턴 (Bad)**: "반복적 개선 없이 초기 아이디어만으로 만족하는 경우"

**Case 4: 점진적 난이도 설계 (Portal)**
- **좋은 예 (Good)**: "초급 퍼즐(기본 포탈 1개 배치, 1~2가지 해결 방법) → 중급(이중 포탈 활용, 중력 파괴, 3~4가지 접근법) → 고급(타이밍 기반 포탈 배치, 충돌 판정 이해 필요, 5가지 이상의 창의적 해결책). 각 단계별 완수 시간: 초급 2분 30초 → 중급 5분 → 고급 10~15분. 플레이어가 느끼는 '아하!' 순간이 각 단계마다 1회 이상 발생하도록 설계"
- **안티패턴 (Bad)**: "단계적 난이도 증가 없이 갑자기 어려워지거나, 모든 퍼즐이 동일 복잡도"

**Case 5: 피드백 기반 AI 훈련 메커니즘 (Little Learning Machines)**
- **좋은 예 (Good)**: "플레이어가 AI 학습 파라미터 조정(신경망 가중치 최대 5개) → AI의 해결 시도 관찰(매초마다 상태 표시: 성공률 0~100%) → 결과 평가(정확도 데이터 그래프 표시) → 5번의 반복 실험으로 목표 달성. 실패 시 '어느 파라미터가 문제인지' 힌트 제공. 최적화 성공 시 학습 곡선 시각화로 보상"
- **안티패턴 (Bad)**: "AI 메커니즘이 불투명하거나 플레이어가 학습 진행률을 추적 불가능"

**Case 6: 시간 제한 퍼즐 (The Witness)**
- **좋은 예 (Good)**: "선 그리기 퍼즐: 정적 버전(시간 제한 없음, 패턴만 제시) → 역동적 버전(20초 시간 제한, 주변 환경과 선이 서로 영향) → 최고난이도(10초 제한 + 여러 선을 동시에 관찰 필요, 직관적 이해 요구). 시간 제한이 있어도 완수 가능한 '최적 경로'가 항상 존재하며, 이는 플레이어 숙련도에 따라 3~5가지 변형으로 제공"
- **안티패턴 (Bad)**: "시간 제한이 퍼즐 해결 자체보다 스트레스를 유발하거나, 무작위 요소로 인해 재현 불가능"

**Case 7: 제시 방식이 난이도를 만든다 (Portal 2 vs The Turing Test)**
- **좋은 예 (Good)**: "사실상 동일한 라이트브리지+버튼 퍼즐에서 Portal 2는 요소를 분리 배치해 발견의 쾌감을 제공"
- **안티패턴 (Bad)**: "The Turing Test는 다리가 이미 버튼 위에 놓여 있어 답이 노출되어 시시해짐 — 힌트 배치는 '사고 재료 제공'까지만, '해법 노출'은 금지"

**Case 8: NPC 힌트 스포일러 (God of War Ragnarok)**
- **안티패턴 (Bad)**: "퍼즐 도착 수십 초 만에 동료 NPC가 답을 말함 — 후반 플레이테스트의 막힘을 레벨 수정 대신 값싼 대사 녹음으로 해결한 결과"
- **좋은 예 (Good)**: "A Plague Tale식 힌트 빈도 설정 옵션 + 긴 지연 시간. 가능하면 대사보다 레이아웃·카메라·시각 명료성 수정이 먼저"

**Case 9: 추리를 객관식으로 만들지 마라 (Return of the Obra Dinn)**
- **좋은 예 (Good)**: "3인 세트가 맞아야만 확정되는 구조로 찍기를 무력화. Ace Attorney는 진술 5개×증거 수십 개의 조합 폭발+오답 페널티로, Her Story는 플레이어가 먼저 검색어를 타이핑해야 하는 자유 입력으로 해결"
- **안티패턴 (Bad)**: "선택지를 보여주는 순간 플레이어는 추리 대신 소거법·찍기를 시작"

## 8. 핵심 인사이트 (Key Insights)

- **팀워크 중심 설계 (Escape Room)**: 퍼즐을 협력적으로 해결하도록 구성하고, 논리적·물리적·사회적 퍼즐의 균형을 유지하세요.
- **다양성 우선 (Jackbox Party Pack)**: 서로 다른 메커닉의 게임들을 제공하여 리스크를 분산하고 창의적 자유를 탐색하세요.
- **명확한 목표 설정**: 플레이어가 무엇을 해야 할지 즉시 이해 가능하도록 직관적 인터페이스와 피드백 루프를 제공하세요.
- **AI 훈련 메커닉 (Little Learning Machines)**: 플레이어가 AI의 학습 과정을 직접 관찰하고 개입할 수 있는 몰입적 경험을 제공하세요.
- **컨텐츠 우선 반복 (Salt and Sanctuary)**: 상상한 경험을 먼저 구현하고 지속적으로 개선하는 접근법을 사용하세요.
- **퍼즐은 설계가 아니라 큐레이션이다 (Jonathan Blow)**: 메카닉 구현 → 파생 현상 전수 탐색 → 흥미로운 현상당 최소 셋업의 퍼즐 1개 → 놀랍지 않은 것은 볼륨이 모자라도 폐기. 목표는 플레이어의 "겨우 풀었다"가 아니라 "이해했다"입니다.
- **요소 수는 최소화하라**: 요소가 많으면 대부분은 소음이고 리셋만 고통스럽습니다. 정답을 아는 순간 실행이 쉬운 구성이 좋은 퍼즐의 전제입니다.
- **낙하물 퍼즐의 게임성은 리스크&리턴이다 (사쿠라이 마사히로)**: 테트리스는 쌓임 자체가 리스크, 지우면 리스크 하락, 모아서 한번에 지우면 키운 리스크만큼의 쾌감 — '지우면 왜 재미있는가'부터 정의하고 기획을 시작하세요. 동사 자체를 갈아끼우는 것(지우기→쏘아 올리기, Meteos)도 유효한 차별화입니다.
- **퍼즐 보스는 규칙 가독성이 난이도보다 우선이다 (Trauma Center)**: 기믹의 인과가 게임 안에서 설명되지 않으면 플레이어는 '운으로 이겼다'고 느끼고 치즈 전략으로 도피합니다.
- **막힘 대응의 우선순위**: 레이아웃·카메라·시각 명료성 수정 → 지연된 힌트 → 옵션화된 직접 힌트 순서로 해결하세요. 완주율을 위해 발견의 쾌감을 지불하는 것은 최후의 수단입니다.
- **어드벤처 퍼즐의 3원칙**: (1) 장·단기 목표를 항상 알게 하고 막힘 대비로 동시 목표를 여러 개 열어둘 것(Day of the Tentacle) (2) 대사·오브젝트 조사에 단서를 흩뿌릴 것 (3) 오답에 이유와 실마리를 줄 것. 3원칙 위반의 대표 사례가 Gabriel Knight 3의 '고양이 털 콧수염' 퍼즐입니다.
- **첫 시도는 실패하게 설계하라 (Stephen's Sausage Roll)**: 플레이어가 당연하다고 믿는 풀이로 유도해 첫 시도를 실패시키는 '가정(함정)'이 이후의 계시를 만듭니다. 첫 시도에 풀리는 퍼즐은 퍼즐이 아니라 절차입니다.
