# UI 메뉴 & 플로우 작성 가이드

## 1. 목적 (Purpose)
게임의 UI 메뉴와 플로우를 체계적으로 설계하여 플레이어가 원하는 기능에 빠르고 직관적으로 접근할 수 있도록 하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 메뉴 구조, 네비게이션 플로우, 화면 전환, 사용자 경로, 백 버튼 처리
- **제외**: 구체적인 그래픽 디자인, 애니메이션 타이밍, 코드 구현

## 3. 설계 목표 (Design Goals)

1. **빠른 접근 (Quick Access)**: 플레이어가 원하는 기능에 최소 클릭으로 도달할 수 있도록 합니다.
2. **명확한 네비게이션 (Clear Navigation)**: 현재 위치와 이동 경로가 항상 명확합니다.
3. **일관된 패턴 (Consistent Patterns)**: 게임 전체에서 일관된 메뉴 패턴을 유지합니다.
4. **에러 방지 (Error Prevention)**: 실수로 인한 중요 행동을 방지하는 확인 절차를 제공합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 메뉴 구조 (Menu Structure)
- **메인 메뉴**: 게임 시작, 설정, 종료 등
- **인게임 메뉴**: 일시정지, 인벤토리, 맵 등

### 4.2 네비게이션 플로우 (Navigation Flow)
- **화면 전환**: 메뉴 간 이동 경로
- **백 버튼**: 이전 화면으로 돌아가기

### 4.3 사용자 경로 (User Paths)
- **주요 시나리오**: 일반적인 사용 경로
- **예외 처리**: 에러 상황 대응

### 4.4 선택 및 심화 (Optional & Advanced)
- **프로토타이핑 도구**: Figma/Adobe XD로 플로우 프로토타입 제작, 클릭 가능한 목업으로 조기 테스트, 플레이어 피드백 반영
- **메뉴 성능 최적화**: 메뉴 열기 시간 0.3초 이내 목표, 대용량 인벤토리 가상 스크롤 적용, 메뉴 프리로딩 전략
- **접근성 고려**: 컨트롤러/키보드/터치 모든 입력 지원, 음성 내레이션 지원, 고대비 모드 옵션
- **A/B 테스트**: 메뉴 배치 2-3가지 버전 테스트, 플레이어 클릭 히트맵 분석, 메뉴 이탈률 추적
- **메뉴 상태 관리**: 메뉴 스택 관리 시스템, 뒤로 가기 히스토리 저장, 비정상 종료 시 복구 전략

## 5. 관련 개념 (Key Concepts)

### 5.1 네비게이션 구조
- **Navigation Flow**: 화면 간 이동 경로와 구조
- **Breadcrumb**: 현재 위치를 표시하는 네비게이션 보조 도구
- **Back Button**: 이전 화면으로 돌아가는 기능

### 5.2 사용자 인터페이스
- **Modal Dialog**: 사용자 확인이 필요한 팝업 창
- **Menu Stack**: 메뉴 계층 관리 시스템
- **Loading State**: 화면 전환 시 로딩 상태 표시

### 5.3 설계 & 테스트
- **Wireframe**: UI 구조 설계 프레임워크
- **Clickable Prototype**: 클릭 가능한 프로토타입
- **Heatmap Analysis**: 플레이어 클릭 패턴 분석

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 메뉴 구조
- [ ] 모든 주요 기능이 3클릭 이내에 접근 가능한가?
- [ ] 메뉴 계층 구조가 논리적인가?
- [ ] 자주 사용하는 기능이 빠르게 접근 가능한가?

### 6.2 네비게이션
- [ ] 현재 위치가 항상 명확하게 표시되는가?
- [ ] 백 버튼이 일관되게 작동하는가?
- [ ] 화면 전환이 자연스러운가?

### 6.3 에러 방지
- [ ] 중요한 행동에 확인 대화상자가 제공되는가?
- [ ] 실수로 인한 데이터 손실을 방지하는가?
- [ ] 모든 경로가 테스트되었는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 빠른 접근**
- **좋은 예 (Good)**: "메인 메뉴 → 설정 → 오디오 (2클릭), 인게임 → 인벤토리 (1클릭), 자주 사용하는 기능을 상위 메뉴에 배치"
- **안티패턴 (Bad)**: "기본 기능에 접근하기 위해 5단계 이상의 메뉴를 거쳐야 함"

**Case 2: 명확한 네비게이션**
- **좋은 예 (Good)**: "현재 메뉴 이름을 상단에 표시, 브레드크럼으로 경로 표시, 백 버튼이 항상 동일한 위치에 배치"
- **안티패턴 (Bad)**: "현재 위치를 알 수 없고, 백 버튼이 화면마다 다른 위치에 배치"

**Case 3: 에러 방지**
- **좋은 예 (Good)**: "게임 종료 시 '저장하지 않은 데이터가 있습니다. 종료하시겠습니까?' 확인, 중요 아이템 판매 시 재확인"
- **안티패턴 (Bad)**: "확인 없이 즉시 실행되어 실수로 인한 데이터 손실"

**Case 4: The Last of Us Part II - 게임내 메뉴 최적화**
- **구현 사례**: "메인 메뉴 → 인벤토리 1클릭, 백팩 관리 화면에서 아이템 선택 후 사용/버리기 2클릭 이내, 맵은 언제든 오른쪽 D패드 호출 가능, 현재 위치/목표가 맵 상단에 실시간 표시, 스토리 진행 시 자동 보관소 기능으로 인벤토리 관리 자동화"
- **핵심 성과**: "전투 중 아이템 접근 최소화(0.5초 이내), 인벤토리 화면 진입 시간 0.3초 목표 달성, 플레이어 상황 인지도 95% 이상"

**Case 5: Forza Horizon 5 - 모듈식 메뉴 구조**
- **구현 사례**: "메인 메뉴의 각 기능이 4개 주요 탭(자동차, 레이스, 튜닝, 멀티플레이)으로 구성, 각 탭 내부는 최대 2단계 깊이 유지, 자동차 선택 → 튜닝 직진로 3클릭으로 단축, 자주 사용하는 '빠른 레이스' 아이콘을 메인 화면에 고정"
- **핵심 성과**: "메뉴 이동 평균 1.8클릭으로 단축, 플레이어 메뉴 이탈률 12% 감소, 빠른 경기 시작 시간 평균 45초로 개선"

**Case 6: Celeste - 미니멀 UI와 명확한 네비게이션**
- **구현 사례**: "메인 메뉴는 4개 옵션만 제공(시작, 계속, 설정, 종료), 화면 좌측에 항상 '뒤로가기' 버튼 고정, 설정 메뉴는 3개 카테고리(기본, 조작, 그래픽)로 단순화, 각 옵션 선택 시 즉시 적용되고 백버튼으로 이전 상태 유지, 세이브 데이터 손실 방지를 위해 삭제 전 3번 확인"
- **핵심 성과**: "유저 인터페이스 학습 곡선 최소화(2분 이내 모든 메뉴 숙달), 플레이테스트 피드백 긍정 응답 98%, 메뉴 혼란 보고 0건"

## 8. 핵심 인사이트 (Key Insights)

- **3클릭 규칙**: 모든 주요 기능이 3클릭 이내에 접근 가능하도록 설계하여 플레이어 편의성을 극대화하세요.
- **일관된 패턴**: 백 버튼, 확인 버튼 등이 항상 동일한 위치와 동작으로 작동하도록 하세요.
- **명확한 피드백**: 화면 전환 시 로딩 상태를 표시하고, 현재 위치를 명확히 전달하세요.
- **에러 방지 우선**: 중요한 행동에는 항상 확인 절차를 제공하여 실수를 방지하세요.
