# QA 계획 & 테스트 케이스 작성 가이드

## 1. 목적 (Purpose)
게임의 QA 계획과 테스트 케이스를 체계적으로 작성하여 출시 전 품질을 보장하고 버그를 최소화하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: QA 전략, 테스트 케이스, 테스트 주기, 버그 트래킹, 자동화 테스트
- **제외**: 구체적인 테스트 도구 구현, 성능 테스트 세부사항

## 3. 설계 목표 (Design Goals)

1. **높은 품질 (High Quality)**: 출시 전 모든 주요 버그를 발견하고 수정합니다.
2. **효율적인 테스트 (Efficient Testing)**: 제한된 시간과 리소스로 최대 커버리지를 달성합니다.
3. **자동화 (Automation)**: 반복적인 테스트를 자동화하여 효율성을 높입니다.
4. **명확한 기준 (Clear Criteria)**: 출시 기준이 명확하고 측정 가능합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 QA 전략 (QA Strategy)
- **테스트 범위**: 기능·성능·호환성
- **테스트 주기**: 일일·주간·마일스톤

### 4.2 테스트 케이스 (Test Cases)
- **기능 테스트**: 모든 기능 검증
- **회귀 테스트**: 수정 후 재검증
- **사용자 시나리오**: 실제 플레이 경로

### 4.3 버그 관리 (Bug Management)
- **버그 우선순위**: Critical·High·Medium·Low
- **버그 트래킹**: 발견·수정·검증

### 4.4 선택 및 심화 (Optional & Advanced)
- **자동화 테스트**: 빌드 배포 시 자동 스모크 테스트(Smoke Test) 스크립트, 주요 API 응답 속도 모니터링
- **호환성 테스트**: OS 버전별(Windows/Android/iOS) 호환성 매트릭스, 최소/권장 사양 기기 리스트
- **데이터 검증**: 대규모 더미(Dummy) 데이터 생성 후 DB 로드 테스트, 밸런스 시뮬레이션 결과 검증

## 5. 관련 개념 (Key Concepts)

- **Test Case**: 특정 기능을 검증하는 테스트 시나리오
- **Regression Testing**: 수정 후 기존 기능 재검증
- **Bug Priority**: 버그의 심각도와 우선순위
- **Test Automation**: 자동화된 테스트

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 QA 전략
- [ ] 테스트 범위가 명확하게 정의되었는가?
- [ ] 테스트 주기가 개발 일정과 동기화되었는가?
- [ ] QA 팀의 역할이 명확한가?

### 6.2 테스트 케이스
- [ ] 모든 주요 기능에 테스트 케이스가 있는가?
- [ ] 테스트 케이스가 명확하고 재현 가능한가?
- [ ] 엣지 케이스가 고려되었는가?

### 6.3 버그 관리
- [ ] 버그 우선순위 기준이 명확한가?
- [ ] 버그 트래킹 시스템이 구축되었는가?
- [ ] Critical 버그의 수정 기한이 정의되었는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 테스트 케이스**
- **좋은 예 (Good)**: "TC001: 플레이어 점프 - 전제조건: 평지에 서 있음 / 단계: 점프 버튼 누름 / 예상 결과: 캐릭터가 1m 높이로 점프, 명확하고 재현 가능"
- **안티패턴 (Bad)**: "점프 테스트 - 점프가 작동하는지 확인" (모호함)

**Case 2: 버그 우선순위**
- **좋은 예 (Good)**: "Critical: 게임 크래시 (24시간 내 수정) / High: 주요 기능 불가 (3일 내) / Medium: 밸런스 문제 (1주 내) / Low: UI 오타 (출시 전), 명확한 기준"
- **안티패턴 (Bad)**: "모든 버그를 동일하게 취급하여 Critical 버그 방치"

**Case 3: 자동화 테스트**
- **좋은 예 (Good)**: "유닛 테스트: 핵심 로직 검증 / 통합 테스트: 시스템 간 상호작용 / 회귀 테스트: 매일 자동 실행, 효율적인 테스트"
- **안티패턴 (Bad)**: "모든 테스트를 수동으로 진행하여 시간 낭비"

## 8. 핵심 인사이트 (Key Insights)

- **테스트 자동화**: 반복적인 테스트를 자동화하여 QA 효율성을 극대화하세요.
- **명확한 우선순위**: Critical 버그는 24시간 내 수정하는 등 명확한 기준을 수립하세요.
- **조기 테스트**: 개발 초기부터 테스트를 시작하여 버그를 조기에 발견하세요.
- **실제 사용자 시나리오**: 실제 플레이어가 경험할 시나리오를 테스트하여 실용적인 품질을 보장하세요.
