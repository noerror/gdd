# UX 전체 원칙 & 사용성 규칙 작성 가이드

## 1. 목적 (Purpose)
게임의 UX 전체 원칙과 사용성 규칙을 일관된 기준으로 작성하여 플레이어 경험의 품질을 보장하고 인지 부하를 최소화하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 휴리스틱 평가, 일관성 원칙, 에러 방지, 가시성, 피드백 시스템, 접근성 요구사항
- **제외**: 구체적인 UI 시각 디자인, 플랫폼별 구현 세부사항, 비주얼 스타일 가이드

## 3. 설계 목표 (Design Goals)

1. **인지 부하 최소화 (Minimize Cognitive Load)**: 플레이어의 정신적 노력을 줄여 게임에 집중할 수 있도록 합니다.
2. **직관적 상호작용 (Intuitive Interaction)**: 플레이어가 자연스럽게 게임을 이해하고 사용할 수 있는 인터페이스를 제공합니다.
3. **감정적 참여 (Emotional Engagement)**: 플레이어의 감정적 몰입을 유도하여 장기적 참여를 증진합니다.
4. **포용성 디자인 (Inclusive Design)**: 다양한 능력과 배경을 가진 플레이어를 고려한 접근성을 보장합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 핵심 원칙 (Core Principles)
- **시스템 상태 가시성**: 플레이어에게 현재 상태를 명확히 전달
- **일관성과 표준**: 플랫폼·장르 표준 준수
- **에러 방지**: 실수 발생 가능성 최소화

### 4.2 사용성 규칙 (Usability Rules)
- **사용자 컨트롤과 자유**: 플레이어의 실수로부터 보호
- **유연성과 효율성**: 숙련자와 초보자의 다양한 사용성 지원
- **미학적·미니멀 디자인**: 불필요한 요소 제거

### 4.3 접근성 & 피드백 (Accessibility & Feedback)
- **접근성 요구사항**: 색각·모터·인지 접근성
- **피드백 시스템**: 행동 → 결과 → 학습의 순환

### 4.4 선택 및 심화 (Optional & Advanced)
- **플랫폼 대응**: PC(키보드/마우스)와 콘솔(패드), 모바일(터치) 조작계별 UI 레이아웃 및 UX 플로우 분기
- **글로벌 대응**: 다국어 텍스트 길이 가변성(독일어 등) 대응, RTL(아랍어) 레이아웃 반전, 폰트 폴백(Fallback)
- **접근성 심화**: 색각 이상 모드, TTS(Text-to-Speech), 한 손 조작 지원, UI 크기/투명도 조절 옵션

## 5. 관련 개념 (Key Concepts)

### UX 원칙
- **Cognitive Load**: 플레이어의 정신적 처리 용량
- **Heuristic Evaluation**: 전문가 기반 사용성 평가 방법
- **Emotional Design**: 플레이어 감정을 고려한 디자인 접근

### 접근성
- **Color Accessibility**: 색각 이상자가 정보를 구분할 수 있는 디자인
- **Motor Accessibility**: 운동 장애가 있는 플레이어를 위한 입력 옵션
- **Cognitive Accessibility**: 다양한 인지 능력을 고려한 정보 제시

### 인지·조작
- **Affordance**: 형태가 기능을 암시하는 신호. 컨트롤(트리거=페달)·내비게이션(길·문·틈)·환경(맥락 배치)·시스템(이름이 상성을 직관시킴)의 4층위로 점검
- **False Affordance**: 시스템과 어긋난 인지 신호(도구 외형 등). 올바른 학습을 2배로 어렵게 만듦
- **Working Memory Limit**: 학습 상황에서 동시 처리 가능한 정보는 평균 3항목
- **Choice Paralysis**: 선택 행위 자체가 부담이 되는 상태. 디폴트 제시·선택 분할·정보 계층화로 대응

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 핵심 원칙
- [ ] 각 UX 원칙이 Nielsen의 10가지 휴리스틱을 기반으로 하는가?
- [ ] 게임의 장르와 플랫폼에 적합한 원칙인가?
- [ ] 일관성 원칙이 게임 전체에 적용 가능한가?

### 6.2 사용성
- [ ] 플레이어의 인지 능력(주의력, 기억력, 학습)을 고려했는가?
- [ ] 에러 방지 및 복구 메커닉이 명시되어 있는가?
- [ ] 감정적 참여를 위한 피드백 메커닉이 정의되어 있는가?

### 6.3 접근성
- [ ] 접근성 요구사항이 포함되어 있는가?
- [ ] 색각·모터·인지 접근성이 고려되었는가?
- [ ] 프로토타입이나 테스트 결과를 바탕으로 검증되었는가?

### 6.4 속도·마찰
- [ ] 로고·오프닝·이벤트 신을 스킵(가능하면 일괄)할 수 있는가?
- [ ] 무비를 일시정지할 수 있는가?
- [ ] 반복 빈도 최상위 행동의 입력 수가 최소화되어 있는가?
- [ ] 로딩 화면 팁이 동시 3개 이하인가?
- [ ] 전투 중 필수 정보가 화면 구석이 아닌 중앙/게임 오브젝트 근처에 표시되는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 시스템 상태 가시성**
- **좋은 예 (Good)**: "플레이어 체력을 0-100% 범위의 시각적 게이지로 표시, 낮은 체력(30% 이하) 시 빨간색 경고 애니메이션, 체력 회복 시 즉각적인 시각·청각 피드백"
- **안티패턴 (Bad)**: "체력이 텍스트로만 표시되어 시각적 긴급성을 알기 어려움, 회복 시 피드백 없음"

**Case 2: 에러 방지**
- **좋은 예 (Good)**: "중요 아이템 판매 시 확인 대화상자, 되돌리기 옵션 제공, 실수로 인한 게임 진행 불가 상황 방지"
- **안티패턴 (Bad)**: "중요 아이템을 실수로 판매할 수 있는 확인 절차 없음"

**Case 3: 접근성 (Ian Hamilton)**
- **좋은 예 (Good)**: "색각 접근성(색상 외 패턴·아이콘 사용), 모터 접근성(재매핑 가능한 컨트롤), 인지 접근성(명확한 정보 계층)"
- **안티패턴 (Bad)**: "색상만으로 정보 구분, 특정 그룹 배제"

**Case 4: 인지 부하 최소화 (Celeste)**
- **좋은 예 (Good)**: "간결한 UI(체력·대시 게이지만 표시), 핵심 메커닉에 집중, 어시스트 모드로 난이도 조절, 플레이어가 실수를 자연스럽게 학습"
- **안티패턴 (Bad)**: "화면을 가득 채운 복잡한 UI로 인지 부하 과다"

**Case 5: 일관성과 피드백 (Horizon Zero Dawn)**
- **좋은 예 (Good)**: "모든 인터랙션에 일관된 버튼 프롬프트, 행동마다 즉각적인 시각·청각 피드백, 포커스 모드로 환경 정보 명확히 표시"
- **안티패턴 (Bad)**: "화면마다 다른 조작 방식, 피드백 부재로 행동 결과 불명확"

**Case 6: 유연성과 효율성 (God of War 2018)**
- **좋은 예 (Good)**: "초보자: 스토리 모드, 숙련자: God 난이도, 컴뱃 깊이 제공, 퀵타임 이벤트 간소화로 접근성 향상"
- **안티패턴 (Bad)**: "한 가지 난이도만 제공하여 초보자 또는 숙련자 배제"

**Case 7: 빈번 작업 접근 비용 (Ocarina of Time vs DOOM 2016)**
- **좋은 예 (Good)**: "DOOM 2016 - '마지막 무기로 전환'을 휠 버튼 탭 한 번에 배정하여 전투 리듬 유지"
- **안티패턴 (Bad)**: "Ocarina of Time(N64) - 아이언 부츠가 장비 메뉴 깊숙이 있어 물의 신전 내내 메뉴 왕복 강제, 던전 악명의 주 원인(3DS판에서 핫키화로 수정)"

**Case 8: 선택 마비 대응 (CRPG 캐릭터 생성)**
- **좋은 예 (Good)**: "프리셋/디폴트 빌드를 전면에 배치, 큰 선택 하나를 여러 시점의 작은 선택으로 분할, 세부 스탯 수십 개는 대표 스탯 1개 + 마우스오버 상세로 계층화"
- **안티패턴 (Bad)**: "스탯이 가득한 생성 화면을 첫 관문으로 배치 — 입문자가 게임 시작 전에 이탈"

## 8. 핵심 인사이트 (Key Insights)

- **인지 처리 한계 (Celia Hodent)**: 플레이어의 인지 처리 한계를 고려한 디자인이 필요하며, 인지 부하 관리·학습 곡선 최적화·감정적 참여 유도를 통합하세요.
- **포용성 디자인 (Ian Hamilton)**: 장애는 환경과의 불일치에서 발생하며, 디자인으로 해결 가능하므로 색각·모터·인지 접근성을 고려하세요.
- **UX 균형 (Ian Livingstone)**: UX는 창의성, 비즈니스, 과학의 균형이 필요하며, 사용자 연구와 디자인의 통합적 접근을 추구하세요.
- **Nielsen 휴리스틱**: 10가지 휴리스틱을 게임 특화 원칙으로 적용하고, 측정 가능한 평가 기준을 수립하세요.

### 인지 한계 기반 원칙
- **작업기억 동시 3항목 (Celia Hodent)**: 학습 모드에서 동시 처리 가능한 정보는 평균 3항목입니다. 로딩 화면 팁은 3개 이하로 제한하고, '어떻게'가 아니라 '왜'를 설명하세요 — 개발자는 지식의 저주 때문에 디테일을 과잉 설명합니다.
- **멀티태스킹은 신화**: 시선 추적 실측에서 전투 중인 플레이어의 시선은 화면 상단의 긴급 경고를 한 번도 향하지 않았습니다(Fortnite 알파). 전투 중 화면 구석의 정보는 없는 것과 같으므로 긴급 정보는 화면 중앙 또는 게임 오브젝트 위에 배치하세요.
- **화면 중앙 응시와 시각 중력**: 플레이어는 HUD를 볼 때를 제외하면 화면 중앙만 응시하며, 하단이 상단보다 주의를 더 받습니다. 인간의 중심와는 팔 뻗은 엄지 폭의 시야만 선명합니다 — 실루엣·명도·구도는 미학이 아니라 플레이 가능성의 문제입니다.
- **초당 요구 판단 수로 복잡도 상한을 계산**: 턴제 수준의 규칙량을 실시간 속도로 요구하면 치명적 복잡도가 됩니다. 시작 전에 규칙 전부를 알아야 하는 '환원 불가능한 복잡도'는 학습 부담을 극대화하므로, 규칙을 단계별로 연습 가능하게 분해할 수 있는지 검사하세요.

### 속도·마찰 제거 (사쿠라이 마사히로)
- **"느림은 죄"**: 게임의 개성은 취향이 갈리지만 '느린 것'만은 만인이 싫어합니다. 게임을 켜둔 시간 중 실제 조작 시간의 비율을 지표로 삼고, 무의미한 공백은 '조금 과하다 싶게' 조이세요. 화면 전환도 성급하다 싶을 정도가 적당합니다.
- **스킵·일시정지 전면 허용**: 로고·오프닝·이벤트 신은 가능하면 일괄 스킵 가능해야 하고, 무비는 반드시 일시정지 가능해야 합니다. 세이브 데이터 부재가 '처음 보는 유저'를 뜻하지 않습니다(2회차, 가족 계정).
- **버튼 연타로 본편 진입**: 문자를 못 읽어도 버튼 연타만으로 기본 메인 게임에 도달하게 설계하세요. 언어·연령 장벽에 대한 최후의 안전망입니다.
- **가장 빈번한 작업을 가장 빠르게**: 반복 빈도 최상위 행동의 입력 수를 최우선으로 최적화하세요. 분류 논리(장비/아이템)보다 사용 빈도가 핫키 배정의 기준입니다.

### 어포던스와 첫인상
- **거짓 어포던스는 학습을 2배로 어렵게**: Fortnite는 채집을 곡괭이 하나로 통합했지만 도끼가 손에 있으면 플레이어는 나무를 찍으려 합니다. 도구 외형과 시스템의 불일치를 제거하자 신규 유저 재방문이 10% 개선됐습니다.
- **"무해한 차이는 없다"**: 지면 문제로 UI 박스 크기만 살짝 줄이자 테스터들이 "아이템 3개 제한이죠?"라고 반복해 물었습니다(Root). 플레이어는 미세한 차이를 반드시 의미로 읽으므로 의도 없는 시각 차이를 남기지 마세요.
- **첫인상 대리 지표는 'UI 청결도'**: 스트리머 실플레이 관찰에 따르면 첫 실행 평가에서 그래픽보다 UI 정돈 상태가 먼저 언급되며 게임 전체 품질의 대리 지표로 쓰입니다. 속편의 UI 개선은 복귀 유저가 1초 만에 감지하는 가장 확실한 '달라졌다' 신호입니다.
- **UI는 마지막 작업이 아니다**: The Last of Us는 출시 8개월 전에야 전담 UI 디자이너가 합류해 핵심 요소가 미결정 상태였습니다. UI 리소스 계획을 프리프로덕션 마일스톤에 포함하세요.
