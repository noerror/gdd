# UI 레이아웃 & 화면 구성 작성 가이드

## 1. 목적 (Purpose)
게임의 UI 레이아웃과 화면 구성을 체계적으로 설계하여 플레이어가 정보를 효율적으로 파악하고 직관적으로 상호작용할 수 있도록 하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 화면 레이아웃 구조, 그리드 시스템, 정보 배치 원칙, 반응형 디자인, 플랫폼별 최적화
- **제외**: 구체적인 그래픽 디자인, 애니메이션 구현, 코드 구현

## 3. 설계 목표 (Design Goals)

1. **직관적 배치 (Intuitive Layout)**: 플레이어가 자연스럽게 정보를 찾을 수 있도록 배치합니다.
2. **시각적 계층 (Visual Hierarchy)**: 중요도에 따라 정보를 명확하게 구분합니다.
3. **플랫폼 최적화 (Platform Optimization)**: 각 플랫폼의 특성에 맞게 레이아웃을 조정합니다.
4. **일관성 유지 (Consistency)**: 게임 전체에서 일관된 레이아웃 패턴을 유지합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 레이아웃 구조 (Layout Structure)
- **그리드 시스템**: 기본 그리드, 여백, 정렬
- **화면 영역**: 안전 영역, 액션 영역, UI 영역

### 4.2 정보 배치 (Information Placement)
- **우선순위**: 핵심·보조·부가 정보 배치
- **가독성**: 폰트 크기, 색상 대비, 간격

### 4.3 플랫폼 대응 (Platform Adaptation)
- **반응형 디자인**: 해상도별 조정
- **플랫폼 특화**: PC·콘솔·모바일 최적화

### 4.4 선택 및 심화 (Optional & Advanced)
- **반응형 대응**: 다양한 화면비(21:9, 4:3) 및 노치(Notch) 디자인 대응 안전 영역(Safe Area)
- **입력 전환**: 컨트롤러 연결 시 UI 프롬프트 자동 전환, 터치 제스처 영역 정의
- **최적화**: UI 텍스처 아틀라스(Atlas) 관리, 리드로우(Redraw) 영역 최소화

## 5. 관련 개념 (Key Concepts)

- **Grid System**: 일관된 배치를 위한 격자 구조
- **Visual Hierarchy**: 정보의 중요도에 따른 시각적 우선순위
- **Safe Zone**: 모든 플랫폼에서 보장되는 화면 영역
- **Responsive Design**: 다양한 화면 크기에 대응하는 디자인

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 레이아웃
- [ ] 그리드 시스템이 일관되게 적용되었는가?
- [ ] 안전 영역 내에 모든 중요 정보가 배치되었는가?
- [ ] 시각적 계층 구조가 명확한가?

### 6.2 가독성
- [ ] 폰트 크기가 모든 플랫폼에서 읽기 쉬운가?
- [ ] 색상 대비가 충분한가?
- [ ] 정보 간 간격이 적절한가?

### 6.3 플랫폼
- [ ] 각 플랫폼의 특성이 고려되었는가?
- [ ] 반응형 디자인이 적용되었는가?
- [ ] 모든 해상도에서 테스트되었는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 그리드 시스템**
- **좋은 예 (Good)**: "12열 그리드 시스템, 16px 기본 간격, 모든 UI 요소가 그리드에 정렬, 일관된 여백 유지"
- **안티패턴 (Bad)**: "임의의 배치로 일관성 부족, 정렬되지 않은 요소들"

**Case 2: 시각적 계층**
- **좋은 예 (Good)**: "핵심 정보(체력, 탄약)는 큰 크기와 높은 대비, 보조 정보(미니맵)는 중간 크기, 부가 정보(툴팁)는 작은 크기"
- **안티패턴 (Bad)**: "모든 정보가 동일한 크기와 색상으로 중요도 구분 불가"

**Case 3: 플랫폼 최적화**
- **좋은 예 (Good)**: "PC: 마우스 호버 지원, 콘솔: 컨트롤러 네비게이션 최적화, 모바일: 터치 영역 확대(최소 44px)"
- **안티패턴 (Bad)**: "모든 플랫폼에 동일한 레이아웃 적용으로 사용성 저하"

## 8. 핵심 인사이트 (Key Insights)

- **그리드 시스템 활용**: 일관된 그리드 시스템을 적용하여 정렬과 여백을 유지하고, 시각적 조화를 달성하세요.
- **시각적 계층 구조**: 중요도에 따라 크기·색상·위치를 차별화하여 플레이어가 정보를 빠르게 파악하도록 하세요.
- **플랫폼 특성 고려**: 각 플랫폼의 입력 방식과 화면 크기에 맞게 레이아웃을 최적화하세요.
- **안전 영역 준수**: 모든 중요 정보를 안전 영역 내에 배치하여 모든 플랫폼에서 가시성을 보장하세요.
