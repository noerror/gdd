# UI 레이아웃 & 화면 구성 작성 가이드

## 1. 목적 (Purpose)
게임의 UI 레이아웃과 화면 구성을 체계적으로 설계하여 플레이어가 정보를 효율적으로 파악하고 직관적으로 상호작용할 수 있도록 하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 화면 레이아웃 구조, 그리드 시스템, 정보 배치 원칙, 반응형 디자인, 플랫폼별 최적화
- **제외**: 구체적인 그래픽 디자인, 애니메이션 구현, 코드 구현

## 3. 설계 목표 (Design Goals)

1. **직관적 배치 (Intuitive Layout)**: 플레이어가 자연스럽게 정보를 찾을 수 있도록 배치합니다.
2. **시각적 계층 (Visual Hierarchy)**: 중요도에 따라 정보를 명확하게 구분합니다.
3. **플랫폼 최적화 (Platform Optimization)**: 각 플랫폼의 특성에 맞게 레이아웃을 조정합니다.
4. **일관성 유지 (Consistency)**: 게임 전체에서 일관된 레이아웃 패턴을 유지합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 레이아웃 구조 (Layout Structure)
- **그리드 시스템**: 기본 그리드, 여백, 정렬
- **화면 영역**: 안전 영역, 액션 영역, UI 영역

### 4.2 정보 배치 (Information Placement)
- **우선순위**: 핵심·보조·부가 정보 배치
- **가독성**: 폰트 크기, 색상 대비, 간격

### 4.3 플랫폼 대응 (Platform Adaptation)
- **반응형 디자인**: 해상도별 조정
- **플랫폼 특화**: PC·콘솔·모바일 최적화

### 4.4 선택 및 심화 (Optional & Advanced)
- **반응형 대응**: 다양한 화면비(21:9, 4:3) 및 노치(Notch) 디자인 대응 안전 영역(Safe Area)
- **입력 전환**: 컨트롤러 연결 시 UI 프롬프트 자동 전환, 터치 제스처 영역 정의
- **최적화**: UI 텍스처 아틀라스(Atlas) 관리, 리드로우(Redraw) 영역 최소화

## 5. 관련 개념 (Key Concepts)

- **Grid System**: 일관된 배치를 위한 격자 구조
- **Visual Hierarchy**: 정보의 중요도에 따른 시각적 우선순위
- **Safe Zone**: 모든 플랫폼에서 보장되는 화면 영역
- **Responsive Design**: 다양한 화면 크기에 대응하는 디자인
- **Color Coding**: 메뉴 카테고리를 색으로 구획해 현재 위치를 색으로 인지시키는 기법
- **Skeuomorphism**: 실물 은유(가방·케이스)로 시스템 규칙을 외형으로 선언하는 UI 스타일
- **Radial Menu**: 아날로그 스틱의 방향 선택 기반 원형 메뉴. 16개 내외가 실용 한계

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 레이아웃
- [ ] 그리드 시스템이 일관되게 적용되었는가?
- [ ] 안전 영역 내에 모든 중요 정보가 배치되었는가?
- [ ] 시각적 계층 구조가 명확한가?

### 6.2 가독성
- [ ] 폰트 크기가 모든 플랫폼에서 읽기 쉬운가?
- [ ] 색상 대비가 충분한가?
- [ ] 정보 간 간격이 적절한가?
- [ ] 같은 화면에 여러 색 계열이 혼용되지 않는가?
- [ ] 아이콘이 실사용 최소 스케일에서 서로 구분되는가?
- [ ] 아이템 최대 보유 개수 기준으로 리스트 UI를 스트레스 테스트했는가?
- [ ] 동시 발화 자막의 우선순위/큐잉 규칙이 정의되어 있는가?
- [ ] 시네마틱 중 마우스 커서가 자동으로 숨겨지는가?

### 6.3 플랫폼
- [ ] 각 플랫폼의 특성이 고려되었는가?
- [ ] 반응형 디자인이 적용되었는가?
- [ ] 모든 해상도에서 테스트되었는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 그리드 시스템**
- **좋은 예 (Good)**: "12열 그리드 시스템, 16px 기본 간격, 모든 UI 요소가 그리드에 정렬, 일관된 여백 유지"
- **안티패턴 (Bad)**: "임의의 배치로 일관성 부족, 정렬되지 않은 요소들"

**Case 2: 시각적 계층**
- **좋은 예 (Good)**: "핵심 정보(체력, 탄약)는 큰 크기와 높은 대비, 보조 정보(미니맵)는 중간 크기, 부가 정보(툴팁)는 작은 크기"
- **안티패턴 (Bad)**: "모든 정보가 동일한 크기와 색상으로 중요도 구분 불가"

**Case 3: 플랫폼 최적화**
- **좋은 예 (Good)**: "PC: 마우스 호버 지원, 콘솔: 컨트롤러 네비게이션 최적화, 모바일: 터치 영역 확대(최소 44px)"
- **안티패턴 (Bad)**: "모든 플랫폼에 동일한 레이아웃 적용으로 사용성 저하"

**Case 4: 시각적 계층과 정보 배치 (Horizon Zero Dawn)**
- **좋은 예 (Good)**: "핵심 정보(체력, 무기)는 화면 하단 중앙, 보조 정보(미니맵, 퀘스트)는 상단 코너, 포커스 모드로 중요 정보만 하이라이트"
- **안티패턴 (Bad)**: "모든 정보를 화면에 나열하여 플레이어 압도"

**Case 5: 그리드 시스템과 일관성 (God of War 2018)**
- **좋은 예 (Good)**: "8px 그리드 시스템, 모든 메뉴 화면이 동일한 레이아웃 패턴, 아이템 설명은 항상 오른쪽에 표시"
- **안티패턴 (Bad)**: "화면마다 다른 정보 배치로 플레이어 혼란"

**Case 6: 반응형 디자인 (Overwatch)**
- **좋은 예 (Good)**: "21:9 울트라와이드 지원, 4:3 구형 모니터 대응, UI 스케일 조절 옵션(80%-120%), 모든 해상도에서 안전 영역 보장"
- **안티패턴 (Bad)**: "16:9만 지원하여 다른 화면비에서 UI 잘림"

**Case 7: 가독성 기준 반복 개편 (Halo Wars 2)**
- **구현 사례**: "카드 UI를 D패드 커서형(카드가 작아 판독 불가) → 부채꼴 핸드(겹침·선택 불명확) → 4장 수평 핸드+원클릭 확대 뷰로 3차 개편. 판정 기준은 '비용/타입/이름/일러스트가 항상 읽히는가' 하나로 고정"
- **핵심 성과**: "콘솔 RTS에서 카드 게임의 판독성과 조작 속도를 동시 확보 — 단일 가독성 기준 없는 개편은 표류한다"

**Case 8: 대량 아이템 리스트 (Zelda: Echoes of Wisdom)**
- **안티패턴 (Bad)**: "100개 이상의 소환물(에코)을 단일 수평 리스트로 제공 — 스크롤이 싫어 같은 에코만 쓰는 역효과 관찰"
- **교훈**: "'비효율이 재발견을 유도한다'는 마찰 디자인을 채택하려면, 그 마찰이 의도한 행동을 실제로 낳는지 플레이테스트로 확인. 개선 우선순위: 홀드 가속 → 그룹핑 → 탭+그리드 → 래디얼 → 즐겨찾기"

**Case 9: 규칙을 선언하는 인벤토리 (Resident Evil 4)**
- **좋은 예 (Good)**: "아타셰 케이스 - 큰 아이템이 물리적으로 더 많은 칸을 차지하는 배치로 '공간은 유한하다'는 서바이벌 규칙을 외형 자체로 전달"
- **안티패턴 (Bad)**: "숫자 무게 제한만 표시 — '종이클립 100개까지 소지 가능' 같은 몰입 붕괴 리스크"

## 8. 핵심 인사이트 (Key Insights)

- **그리드 시스템 활용**: 일관된 그리드 시스템을 적용하여 정렬과 여백을 유지하고, 시각적 조화를 달성하세요.
- **시각적 계층 구조**: 중요도에 따라 크기·색상·위치를 차별화하여 플레이어가 정보를 빠르게 파악하도록 하세요.
- **플랫폼 특성 고려**: 각 플랫폼의 입력 방식과 화면 크기에 맞게 레이아웃을 최적화하세요.
- **안전 영역 준수**: 모든 중요 정보를 안전 영역 내에 배치하여 모든 플랫폼에서 가시성을 보장하세요.

### 화면 설계 원칙 (사쿠라이 마사히로)
- **명료함과 스타일은 상충한다**: '알기 쉬우면 좋은 UI'로 잘라 말할 수 없습니다. Persona 5는 정돈보다 스타일을 택했고 그것이 그 게임의 정답입니다. 게임의 톤에 맞춰 저울을 정하되, 시선 유도·강조 대소·윈도우화(관련 정보 묶기)·휘도 조절·폰트 선택은 어느 쪽이든 통하는 공통 수단입니다.
- **색으로 구획하되 같은 화면에 색을 섞지 마라**: 카테고리별로 하위 메뉴 전체를 같은 색 베이스로 통일하면 현재 위치를 색으로 인지합니다. 문자보다 그림·색이 정보 처리에서 압도적으로 빠릅니다.
- **중요한 메뉴일수록 아이콘을 크게**: 위→아래 리스트 대신 아이콘의 대소로 중요도·사용 빈도를 표현하면 크기 자체가 액세스를 유도합니다. 단, 아이콘화된 메뉴는 긴 언어로 번역 시 문자 넘침에 주의하세요.
- **폰트와 UI가 타깃층을 바꾼다**: 같은 기능이라도 폰트를 쿨하게 하면 성인용, 귀엽게 하면 라이트층으로 보입니다. 캐릭터 리디자인보다 훨씬 적은 비용으로 세계 분위기를 형성하므로, 가독성을 지키는 선에서 적극 디자인하세요.
- **미니맵만 보고 있다면 화면 설계의 패배**: 맵을 반투명 전체 오버레이로 겹치거나, 메인 화면 자체의 정보량을 올려(주행선 가이드, 적 배치로 방향 전달) 시선을 게임 화면으로 되돌리세요. "플레이어가 화면의 어디를 보며 플레이하는가"를 항상 의식해야 합니다.

### 세부 품질 (스트리머 실플레이 관찰)
- **자막 겹침은 몰입 파괴 1순위**: 군중 동시 발화 시 자막이 겹치면 스토리 손실을 감수하고 자막을 꺼버리는 행동이 관찰됩니다. 동시 발화 자막의 우선순위/큐잉 규칙을 명세하세요.
- **다이어제틱 맵은 '메뉴가 아니라서' 최고 평가**: 종이 지도를 꺼내 드는 방식이 "게임 안에 머문다"는 이유로 극찬받습니다. 정보 확인이 게임 세계를 이탈하지 않게 설계하세요.
- **베타의 임시 UI도 브랜드 인상을 남긴다**: 테스트 빌드라도 프론트엔드 아트는 게임 정체성 판정에 포함됩니다. HUD 크기 슬라이더와 상황부 동적 UI(전투 시에만 표시)는 PC판 기대 사양이 됐습니다.
