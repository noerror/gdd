# 온보딩 & 튜토리얼 UX 작성 가이드

## 1. 목적 (Purpose)
게임의 온보딩과 튜토리얼을 체계적으로 설계하여 신규 플레이어가 게임을 빠르게 이해하고 즐길 수 있도록 하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 튜토리얼 구조, 학습 곡선, 점진적 복잡성, 피드백 시스템, 온보딩 플로우
- **제외**: 구체적인 대사 작성, 그래픽 디자인, 레벨 디자인 세부사항

## 3. 설계 목표 (Design Goals)

1. **빠른 학습 (Quick Learning)**: 플레이어가 핵심 메커닉을 빠르게 이해할 수 있도록 합니다.
2. **점진적 복잡성 (Progressive Complexity)**: 간단한 개념부터 복잡한 메커닉까지 단계적으로 소개합니다.
3. **자연스러운 통합 (Natural Integration)**: 튜토리얼이 게임플레이와 자연스럽게 통합됩니다.
4. **플레이어 주도 (Player-driven)**: 플레이어가 자신의 속도로 학습할 수 있도록 합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 온보딩 플로우 (Onboarding Flow)
- **첫 경험**: 게임 시작 후 첫 5분
- **핵심 메커닉**: 기본 조작과 핵심 시스템

### 4.2 튜토리얼 구조 (Tutorial Structure)
- **단계별 학습**: 기본 → 중급 → 고급
- **피드백 시스템**: 성공·실패 피드백

### 4.3 학습 지원 (Learning Support)
- **도움말**: 상황별 힌트, 재방문 가능한 튜토리얼
- **건너뛰기**: 숙련자를 위한 옵션

### 4.4 첫 실행 시퀀스 (First-Run Flow)
- **부팅 순서**: 로고(스킵 가능) → 필수 설정 선행(밝기·감도·자막·재시작 필요 항목) → 즉시 게임 진입
- **도달 시간 목표**: 부팅부터 첫 조작 가능까지의 시간을 수치 목표로 관리 (예: 90초 이내)
- **복귀·재도전 유저**: 전작/이전 세션 리캡 제공, 2회차부터 모든 시네마틱·대사 스킵 허용
- **온보딩 강도 선택**: 가이드 잠금형 진행과 전체 해제형을 시작 시 선택지로 제공

### 4.5 선택 및 심화 (Optional & Advanced)
- **분석 지표**: 단계별 이탈률(Funnel) 추적, 튜토리얼 완료 소요 시간, 스킵 버튼 클릭율
- **예외 처리**: 튜토리얼 중 접속 종료 시 이어하기(Review) 지원, 필수 아이템 미획득 방지
- **접근성**: 튜토리얼 텍스트 TTS 지원, 조작 가이드 다시보기 기능

## 5. 관련 개념 (Key Concepts)

- **Onboarding**: 신규 플레이어를 게임에 적응시키는 과정
- **Progressive Disclosure**: 정보를 점진적으로 공개하는 방식
- **Learning Curve**: 플레이어의 학습 속도와 난이도 증가율
- **Contextual Tutorial**: 상황에 맞는 튜토리얼 제공. 행동 신호로 무지를 추론해 필요한 플레이어에게만 표시
- **Isolation Principle**: 새 메카닉을 '안전한 도입 → 안전장치 제거 심화 → 기존 메카닉과 조합'의 3단계로 가르치는 원칙
- **NPE (New Player Experience)**: 훅 → 튜토리얼 → 보상(즉시+장기)의 3요소로 설계하는 신규 플레이어 경험
- **8단어 규칙 (George Fan)**: 화면에 8단어가 넘으면 읽지 않는다는 툴팁 작성 기준
- **First-Run Flow**: 부팅부터 첫 조작까지의 시퀀스. '입력 가능 상태 도달 시간'이 핵심 지표

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 온보딩
- [ ] 첫 5분 내에 핵심 메커닉을 경험할 수 있는가?
- [ ] 플레이어가 즉시 재미를 느낄 수 있는가?
- [ ] 온보딩이 게임플레이와 자연스럽게 통합되는가?

### 6.2 튜토리얼
- [ ] 정보가 점진적으로 공개되는가?
- [ ] 각 단계마다 명확한 피드백이 제공되는가?
- [ ] 플레이어가 자신의 속도로 학습할 수 있는가?

### 6.3 학습 지원
- [ ] 상황별 힌트가 제공되는가?
- [ ] 튜토리얼을 건너뛸 수 있는가?
- [ ] 재방문 가능한 도움말이 있는가?

### 6.4 첫 실행·검증
- [ ] 부팅부터 첫 조작까지의 시간이 수치 목표로 정의되어 있는가?
- [ ] 튜토리얼 툴팁·프롬프트가 8단어(1줄) 이내인가?
- [ ] 이미 수행한 행동의 튜토리얼이 자동 이수 처리되는가?
- [ ] 전투 등 긴박 상황에서 튜토리얼 팝업이 차단되는가?
- [ ] 튜토리얼 텍스트와 실제 게임 상태의 동기화를 별도 QA로 검증하는가?
- [ ] 온보딩 구간의 사망·이탈 데이터를 추적하는가?
- [ ] 튜토리얼 종료 지점에 다음 목표(브리지)가 배치되어 있는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 점진적 복잡성**
- **좋은 예 (Good)**: "1단계: 이동만 학습 → 2단계: 점프 추가 → 3단계: 공격 추가 → 4단계: 조합 메커닉, 각 단계마다 실습 기회 제공"
- **안티패턴 (Bad)**: "모든 메커닉을 한 번에 설명하여 플레이어 혼란"

**Case 2: 자연스러운 통합 (Portal)**
- **좋은 예 (Good)**: "튜토리얼이 게임 스토리의 일부로 통합, 플레이어가 튜토리얼임을 인지하지 못하면서 자연스럽게 학습, 점진적 난이도 증가"
- **안티패턴 (Bad)**: "게임과 분리된 튜토리얼 모드로 몰입 저해"

**Case 3: 플레이어 주도**
- **좋은 예 (Good)**: "힌트를 선택적으로 표시, 숙련자는 튜토리얼 건너뛰기 가능, 도움말을 언제든지 다시 볼 수 있음"
- **안티패턴 (Bad)**: "강제 튜토리얼로 숙련자 좌절 유발"

**Case 4: 명시적 튜토리얼 구조 (Dead Space)**
- **게임**: Dead Space (2008)
- **온보딩 구조**:
  - 0-2분: 기본 이동 및 시야 회전 (홀로그래픽 가이드 사용)
  - 2-4분: 무기 장착 메커닉 (인터랙티브 패널)
  - 4-6분: 약한 몬스터(Slasher) 1체와 전투 (환경 안전성 높음)
  - 6-10분: 리소스 관리 (탄약 제한 30발, 약 섭취 튜토리얼)
  - 10-15분: 엘리베이터 시퀀스 중 첫 강한 몬스터 조우 (플레이 스타일 실험)
- **피드백**: 화면 코너에 푸른 글자 가이드 텍스트 점진적 추가
- **완료 기준**: 첫 보스 쓰러뜨리기 (평균 12분 소요)
- **스킵 옵션**: 없음 (엄격한 선형 튜토리얼), 뉴게임+ 모드에서는 완전 생략

**Case 5: 게임 내 컨텍스트 학습 (Celeste)**
- **게임**: Celeste
- **온보딩 설계**:
  - 첫 5분: 대시(공중에서 한 번 방향 점프)만 교육 → 실습 구간 제공
  - 5-10분: 점프 조합 (대시 + 점프 + 벽 점프) 체이닝 실습
  - 10-15분: 새로운 환경 기믹 1가지씩 도입 (스파이크, 회전 기둥)
  - 15-25분: 여러 기믹 복합 구간 (난이도 급상승, 죽음으로 학습)
  - 25분+: 실제 스토리 진행 (튜토리얼 완료 마크 표시)
- **학습 곡선**: 사망률 추이 데이터로 조정
  - 처음 5분: 사망률 10% 이하 (너무 쉬움)
  - 10-15분: 사망률 40-50% (적정 난이도)
  - 20-25분: 사망률 60-70% (도전적)
- **건너뛰기**: 아시스트 모드에서 난이도 조정 및 대시 무제한 옵션 제공

**Case 6: 비선형 점진적 공개 (Portal)**
- **게임**: Portal
- **온보딩 철학**: 게임 스토리와 튜토리얼의 완벽한 통합
  - 1-5분: GLaDOS와의 대화 중 이동 및 시점 조작 학습 (강제 아님)
  - 5-10분: 포탈 건 조작 (한 번에 하나만 발사 → 두 개 동시 관리)
  - 10-20분: 포탈 물리 원리 학습 (속도 보존, 각도 계산)
  - 20-40분: 게임의 핵심 3가지 기법 발견 유도 (플레이어 자율)
    1. 포탈을 이용한 높이 상승
    2. 포탈을 이용한 물체 이동
    3. 터렛 회피 및 제거
- **피드백**: 챔버(방) 통과 시 자동 진행 (강제성 없음)
- **학습 지표**:
  - 튜토리얼 완료율: 98% (숨겨진 튜토리얼로 인식 안 함)
  - 평균 완료 시간: 45분 (매우 자연스러운 진행)
  - 건너뛰기 사용 빈도: 0.2% (게임이 재미 자체)
- **특징**: 플레이어가 "튜토리얼"이라는 개념을 지우고 순수 게임플레이만 경험

**Case 7: 오프닝이 코어 경험을 배반한 사례 (Skyrim)**
- **안티패턴 (Bad)**: "핵심 욕구는 '탐험'인데 인트로는 손이 묶인 수레 이동, 첫 3분에 고유명사 10여 개 투하(전부 필터링됨), 처형 직전 캐릭터 생성 메뉴로 서사 긴장 증발, 게임이 빛나는 오픈월드 순간은 20분 뒤에야 도달"
- **교훈**: "인트로는 그 게임의 코어 인게이지먼트를 정확히 팔아야 한다 — 같은 '이동당하는 인트로'라도 CoD4는 코어(혼돈·세트피스)를 팔았기에 성공"

**Case 8: 튜토리얼 재작업 실측 (Mortal Kombat 11)**
- **구현 사례**: "문제 레슨(이탈률 10배)을 고급 섹션으로 이동, 선형 잠금 폐지로 전 레슨 개방, 6레슨 15연습→54레슨 80연습 세분화, 실패 시 단일 이유+행동 지침 제공, 카테고리 완료 시 메뉴로 강제 복귀시켜 '가서 실제 게임을 해도 된다'는 신호 부여"
- **핵심 성과**: "문제 레슨 완료율 51% → 97%. 라이브 게임(Injustice 2)에서 6개월간 사전 검증 후 이식 — 튜토리얼은 메타가 고정된 완성 게임에서 프로토타이핑"

**Case 9: 상황으로 가르치는 10초 룰 (Half-Life 2)**
- **좋은 예 (Good)**: "'반토막 난 좀비 시체 + 손 닿는 곳의 톱날 + 걸어오는 좀비' 두 장면 10초로 핵심 대처법 전달, 새 적은 '안전한 관찰 → 안전한 연습 → 실전' 3단계로 소개"
- **안티패턴 (Bad)**: "같은 정보를 텍스트 4회+음성 2회로 반복 설명 — 모든 튜토리얼 항목에 '팝업 없이 상황으로 가르칠 수 있는가'를 먼저 질문"

## 8. 핵심 인사이트 (Key Insights)

- **첫 5분의 중요성**: 신규 플레이어의 유지율은 첫 5분에 결정되므로, 즉시 재미를 제공하고 핵심 메커닉을 경험하게 하세요.
- **점진적 공개 (Progressive Disclosure)**: 정보를 한 번에 제공하지 말고, 필요한 시점에 점진적으로 공개하여 인지 부하를 최소화하세요.
- **자연스러운 통합 (Portal)**: 튜토리얼을 게임 스토리와 통합하여 플레이어가 학습 중임을 인지하지 못하게 하세요.
- **플레이어 선택권**: 숙련자를 위한 건너뛰기 옵션과 재방문 가능한 도움말을 제공하세요.

### 격리 원칙과 학습 구조
- **격리 원칙 3단계**: 새 메카닉은 (1)안전한 환경에서 도입 → (2)안전장치를 제거해 심화 → (3)기존 메카닉과 조합의 순서로 가르치세요(Super Mario 3D World의 발판 교습이 교본). Portal은 게임의 90%가 튜토리얼인데도 모든 수업을 도전으로 제시해 튜토리얼로 느껴지지 않습니다.
- **격리 원칙의 한계**: Civilization의 행복도 같은 추상·통계 메카닉은 플레이로 자연 습득시키기 어려워 직접 설명이 낫습니다. 턴제 전략·RPG는 적용 난도가 높음을 온보딩 계획에 반영하세요.
- **격리 '챔버'는 실패한다**: Hitman은 메커닉별 격리 챔버가 메커닉이 결합되는 실제 레벨에서 무용지물이 되자 진짜 규칙으로 도는 미니 샌드박스로 전환했습니다. 플레이어 능력을 속이는 페이크 기능 금지, '튜토리얼' 명칭 대신 '프롤로그' 사용까지 검토하세요.
- **조작 학습의 서열 (사쿠라이 마사히로)**: 긴 설명문 < 무기질 연습장 < 실전 속 텍스트 < 능동적 놀이 순으로 우수합니다. 이동계 신규 스킬은 그 스킬을 써야만 나갈 수 있는 좁은 공간에 가두는 것이 정석이며, 어려운 장르는 자유도를 뺀 앞부분(레일 구간)으로 먼저 진입시키세요.

### 첫 실행 의식 (스트리머 실플레이 관찰)
- **최고 평가 공식 "설정 먼저, 바로 게임"**: 설정을 먼저 하게 두고 즉시 게임에 던지는 구성이 최고 반응을 얻습니다. 부팅 45초 만의 첫 성공 체험에서 호평이 시작된 사례처럼, 인트로 영상보다 '입력 가능 상태 도달 시간'이 첫인상을 만듭니다.
- **문단형 팝업 금지, 행동 직전 1줄**: 이동 중 자연스럽게 뜨는 동적 튜토리얼이 교과서 반응을 얻습니다. 이미 완료한 행동은 자동 이수 처리하고, 전투 중 팝업은 최악의 타이밍입니다. 생존 필수 기능(회복)은 최우선으로 가르치세요.
- **종료 지점에 브리지 목표**: 가이드가 목표를 놓고 사라지면 "이제 뭘 해야 하는지 모르겠다"는 방치감이 됩니다. 튜토리얼 종료 시 다음 목표를 반드시 제시하세요.
- **다단계 절차는 게임 내 체크리스트 필수**: 절차형 콘텐츠(제작·수리·의식)의 단계 리스트가 게임 안에 없으면 플레이어는 외부 영상으로 나갑니다. 규칙 설명은 실행 가능한 순간 단위로 쪼개고 상시 재열람 수단을 제공하세요.

### 실측 데이터 기준
- **성인 유저의 온보딩 예산은 약 10분**: 장르 코어 팬조차 "처음 10분 안에 재미없거나 헷갈리면 환불"을 언급합니다. 성공한 F2P도 첫 1시간에 20%가 떠납니다(Warframe 누적 기준 1시간 생존율 80%).
- **온보딩 중 처벌 금지**: 온보딩 구간에서 자주 죽은 플레이어는 이탈 확률이 유의하게 높았습니다(Fortnite 알파 실측). 하드코어 유저는 초반이 쉬우면 불평하지만 계속 플레이합니다.
- **세그먼트별 커리큘럼**: 캠페인층→라이트층→매니아→PvP 세그먼트에서 한 단계 위 세그먼트가 2.5~3.5배 리텐션을 보입니다. 각 층위가 '지금 필요한 것만' 배우고 게임으로 돌아가게 설계하고, 전 커리큘럼 강제는 이탈을 만듭니다.
- **맥락+의미가 있어야 학습이 인코딩된다**: 로딩 화면 텍스트 교습은 실패했고, "이 문을 직접 편집해야 진행되는" 능동 과제로 바꾸자 학습이 확인됐습니다(Fortnite). 반복은 간격을 띄우고 맥락을 바꿔가며(고정 표적→이동 표적→회피 중 사격) 하세요.
- **'왜'를 사이에 끼워라**: "뭘 하는지는 알겠는데 왜 하는지 모르겠다"가 워크스루 불만의 대부분입니다. 단계 사이에 전략적 이유를 설명하면 같은 내용도 호평으로 바뀝니다(Root 재인쇄 사례).

### 서사·프레이밍 활용
- **튜토리얼에 서사적 이중 임무**: FF7은 튜토리얼 강사를 클라우드로 지정해 조작 학습과 '이 남자는 전문가'라는 인상 심기를 동시에 수행합니다. 세계관 연출로 위장한 학습 구간은 스킵 욕구 자체를 지웁니다.
- **첫 관문은 튜토리얼이 아니라 캐릭터 제작창**: 복잡한 생성 화면에서 이미 이탈이 발생합니다. 프리셋 전면 배치, 필수 단계 수 최소화, 사후 리스펙 허용으로 시작 전 마비를 무력화하세요.
- **프롤로그는 확실히 길거나 확실히 짧게**: 1회차 몰입용 장편과 다회차용 스킵 경로 중 하나를 명확히 택하세요. 애매한 중간 길이가 최악입니다.
