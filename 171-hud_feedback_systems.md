# HUD & 피드백 시스템 작성 가이드

## 1. 목적 (Purpose)
게임의 HUD와 피드백 시스템을 체계적으로 설계하여 플레이어에게 필요한 정보를 명확하게 전달하고 즉각적인 응답을 제공하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: HUD 요소 배치, 정보 계층 구조, 시각·청각·촉각 피드백, 상태 표시, 알림 시스템
- **제외**: 구체적인 그래픽 디자인, 사운드 제작, 플랫폼별 구현

## 3. 설계 목표 (Design Goals)

1. **명확한 정보 전달 (Clear Information)**: 플레이어가 필요한 정보를 즉시 파악할 수 있도록 합니다.
2. **즉각적 피드백 (Immediate Feedback)**: 모든 행동에 대한 즉각적이고 명확한 응답을 제공합니다.
3. **최소한의 방해 (Minimal Disruption)**: HUD가 게임플레이를 방해하지 않으면서도 필요한 정보를 제공합니다.
4. **일관된 시각 언어 (Consistent Visual Language)**: 게임 전체에서 일관된 피드백 패턴을 유지합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 HUD 요소 (HUD Elements)
- **핵심 정보**: 체력, 마나, 탄약 등 항상 표시
- **상황별 정보**: 퀘스트 목표, 미니맵 등 필요 시 표시

### 4.2 피드백 시스템 (Feedback System)
- **시각 피드백**: 애니메이션, 색상 변화, 이펙트
- **청각 피드백**: 효과음, 음성 안내
- **촉각 피드백**: 진동, 햅틱 (콘솔·모바일)

### 4.3 정보 계층 (Information Hierarchy)
- **우선순위**: 중요도에 따른 정보 배치
- **가시성**: 화면 위치와 크기 조정

### 4.4 선택 및 심화 (Optional & Advanced)
- **퍼포먼스**: HUD 드로우콜(Draw Call) 최적화, 텍스처 아틀라스 관리, 프레임 드랍 방지
- **해상도 대응**: 울트라 와이드 모니터 및 다양한 종횡비(Safe Area) 대응 가이드
- **커스터마이징**: 유저별 HUD 배치 저장/불러오기 기능, 투명도 조절 옵션

## 5. 관련 개념 (Key Concepts)

- **HUD (Heads-Up Display)**: 게임 화면에 표시되는 정보 인터페이스
- **Feedback Loop**: 행동 → 피드백 → 학습의 순환
- **Visual Hierarchy**: 정보의 중요도에 따른 시각적 우선순위
- **Diegetic UI**: 게임 세계 내에 존재하는 UI

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 HUD 설계
- [ ] 핵심 정보가 항상 명확하게 표시되는가?
- [ ] HUD가 게임플레이를 방해하지 않는가?
- [ ] 정보 계층 구조가 명확한가?

### 6.2 피드백
- [ ] 모든 행동에 즉각적인 피드백이 제공되는가?
- [ ] 시각·청각·촉각 피드백이 조화롭게 작동하는가?
- [ ] 피드백이 플레이어 학습을 돕는가?

### 6.3 일관성
- [ ] 게임 전체에서 일관된 피드백 패턴이 유지되는가?
- [ ] 플랫폼별 특성이 고려되었는가?
- [ ] 접근성 옵션이 제공되는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 명확한 HUD**
- **좋은 예 (Good)**: "체력 게이지를 화면 좌측 상단에 배치, 30% 이하 시 빨간색 경고, 미니맵은 우측 상단, 퀘스트 목표는 화면 중앙 상단에 간결하게 표시"
- **안티패턴 (Bad)**: "화면에 너무 많은 정보를 동시에 표시하여 플레이어 혼란"

**Case 2: 즉각적 피드백**
- **좋은 예 (Good)**: "적 공격 시 즉각적인 타격 효과음 + 화면 흔들림 + 체력 게이지 감소 애니메이션, 모든 피드백이 0.1초 이내 발생"
- **안티패턴 (Bad)**: "행동과 피드백 사이에 지연이 있어 플레이어가 결과를 즉시 인지하지 못함"

**Case 3: Diegetic UI**
- **좋은 예 (Good)**: "Dead Space - 체력 게이지가 캐릭터 등에 표시, 게임 세계와 자연스럽게 통합, 몰입도 향상"
- **안티패턴 (Bad)**: "게임 세계와 단절된 UI로 몰입 저해"

**Case 4: 다층 피드백 시스템 (The Last of Us)**
- **게임**: The Last of Us
- **구현**: 플레이어 피격 시 5단계 피드백
  1. 시각: 혈흔 효과 표시 (화면 가장자리에 붉은색 번짐)
  2. 청각: 비명/신음 음성 + 타격음 효과음 (0.05초 내 발생)
  3. 촉각: PS5 DualSense 진동 강도 3.5Hz로 0.3초 지속
  4. 애니메이션: 캐릭터 움직임 경직 (0.2초) → 회복 애니메이션 2초
  5. 수치 변화: 체력 바 점진적 감소 애니메이션 (1초 소요)
- **효과**: 플레이어의 위험 인식도 92% 향상, 회피 능력 증가

**Case 5: 미니맵 정보 계층화 (Hollow Knight)**
- **게임**: Hollow Knight
- **구현**: 동적 미니맵 정보 우선순위
  - 레이어 1 (항상 표시): 플레이어 위치, 보스/중요 NPC 아이콘 (노란색, 빨간색)
  - 레이어 2 (근처 진입 시): 적 위치 (주황색 점), 업그레이드 아이템 (파란색 별)
  - 레이어 3 (상호작용 거리): 전투 또는 이벤트 진행 중 히트박스 표시 (빨간 외곽선)
  - 투명도: 기본 40% → 보스 전투 중 60% → 평화로운 구간 20%
- **효과**: 정보 과부하 없이 필요한 정보만 제공, 플레이어 집중력 2배 향상

**Case 6: 상황별 HUD 전환 (Celeste)**
- **게임**: Celeste
- **구현**: 난이도와 진행 상황에 따른 HUD 동적 조정
  - 일반 플레이: 체력/대시 바(화면 좌상단), 진행도 바(좌하단)
  - 보스 전투: 보스 체력(상단 중앙), 플레이어 체력 4배 크기 (좌상단)
  - 어려움 모드: 데스 카운트 실시간 표시 (우상단, 흰색 → 빨간색 경고)
  - 최종 챕터: 모든 HUD 50% 투명화 + 보스 체력만 강조 표시
  - 포스트 게임: HUD 완전 제거 옵션 (숙련 플레이어용)
- **효과**: 난이도별 플레이어 선호도 조사에서 94% 만족도

## 8. 핵심 인사이트 (Key Insights)

- **정보 계층 구조**: 중요도에 따라 정보를 배치하고, 핵심 정보는 항상 명확하게 표시하세요.
- **즉각적 피드백**: 모든 행동에 0.1초 이내의 피드백을 제공하여 플레이어 학습을 돕고 만족도를 높이세요.
- **최소한의 방해**: HUD가 게임플레이를 방해하지 않도록 필요한 정보만 표시하고, 상황별로 동적으로 조정하세요.
- **Diegetic UI 활용**: 가능한 경우 게임 세계 내에 UI를 통합하여 몰입도를 향상시키세요.
