# 분석·텔레메트리·KPI 작성 가이드

## 1. 목적 (Purpose)
게임의 분석, 텔레메트리, KPI를 체계적으로 정의하여 데이터 기반 의사결정을 지원하고 게임 품질을 지속적으로 개선하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: KPI 정의, 텔레메트리 이벤트, 데이터 수집 계획, 분석 대시보드
- **제외**: 구체적인 분석 도구 구현, 데이터베이스 설계, 개인정보 보호 법률

## 3. 설계 목표 (Design Goals)

1. **측정 가능성 (Measurability)**: 모든 중요 지표가 정량적으로 측정 가능합니다.
2. **실행 가능성 (Actionability)**: 데이터가 구체적인 개선 행동으로 이어집니다.
3. **개인정보 보호 (Privacy)**: 플레이어 개인정보를 보호하며 데이터를 수집합니다.
4. **실시간 모니터링 (Real-time Monitoring)**: 중요 지표를 실시간으로 추적합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 KPI 정의 (KPI Definition)
- **핵심 KPI**: DAU, MAU, 리텐션, ARPU 등
- **게임 특화 KPI**: 레벨 완료율, 평균 플레이 시간 등

### 4.2 텔레메트리 (Telemetry)
- **이벤트 정의**: 수집할 이벤트와 파라미터
- **수집 빈도**: 실시간, 배치 등

### 4.3 분석 & 대시보드 (Analytics & Dashboard)
- **분석 계획**: 주요 분석 시나리오
- **대시보드**: 실시간 모니터링 화면

## 5. 관련 개념 (Key Concepts)

- **KPI (Key Performance Indicator)**: 핵심 성과 지표
- **Telemetry**: 게임 내 플레이어 행동 데이터 수집
- **DAU/MAU**: 일간/월간 활성 사용자
- **Retention**: 플레이어 유지율

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 KPI
- [ ] 모든 핵심 KPI가 정의되었는가?
- [ ] KPI가 측정 가능하고 명확한가?
- [ ] 목표 값이 설정되었는가?

### 6.2 텔레메트리
- [ ] 중요 이벤트가 모두 수집되는가?
- [ ] 개인정보 보호가 고려되었는가?
- [ ] 데이터 수집이 게임 성능에 영향을 미치지 않는가?

### 6.3 분석
- [ ] 분석 계획이 구체적인가?
- [ ] 대시보드가 실시간으로 업데이트되는가?
- [ ] 데이터가 의사결정에 활용되는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 핵심 KPI**
- **좋은 예 (Good)**: "DAU: 일간 활성 사용자 수, 목표 10,000명 / D1 리텐션: 40% 목표 / ARPU: 사용자당 평균 수익 $2, 모든 KPI가 측정 가능하고 목표 명확"
- **안티패턴 (Bad)**: "플레이어 만족도 같은 추상적 지표만 정의하여 측정 불가"

**Case 2: 텔레메트리 이벤트**
- **좋은 예 (Good)**: "레벨 시작/완료, 아이템 구매, 플레이어 사망, 각 이벤트마다 타임스탬프·플레이어ID·레벨ID 등 파라미터 포함"
- **안티패턴 (Bad)**: "과도한 데이터 수집으로 성능 저하 및 개인정보 침해"

**Case 3: 실시간 대시보드**
- **좋은 예 (Good)**: "DAU·리텐션·ARPU를 실시간 그래프로 표시, 이상 징후 시 알림, 팀 전체가 접근 가능"
- **안티패턴 (Bad)**: "데이터 수집만 하고 분석·활용하지 않음"

## 8. 핵심 인사이트 (Key Insights)

- **측정 가능한 KPI**: 모든 중요 지표를 정량적으로 측정 가능하게 정의하고, 목표 값을 설정하세요.
- **실행 가능한 데이터**: 데이터 수집이 구체적인 개선 행동으로 이어지도록 분석 계획을 수립하세요.
- **개인정보 보호**: GDPR, CCPA 등 개인정보 보호 법률을 준수하며 데이터를 수집하세요.
- **실시간 모니터링**: 중요 지표를 실시간으로 추적하여 문제를 빠르게 발견하고 대응하세요.
