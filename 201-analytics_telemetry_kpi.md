# 분석·텔레메트리·KPI 작성 가이드

## 1. 목적 (Purpose)
게임의 분석, 텔레메트리, KPI를 체계적으로 정의하여 데이터 기반 의사결정을 지원하고 게임 품질을 지속적으로 개선하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: KPI 정의, 텔레메트리 이벤트, 데이터 수집 계획, 분석 대시보드
- **제외**: 구체적인 분석 도구 구현, 데이터베이스 설계, 개인정보 보호 법률

## 3. 설계 목표 (Design Goals)

1. **측정 가능성 (Measurability)**: 모든 중요 지표가 정량적으로 측정 가능합니다.
2. **실행 가능성 (Actionability)**: 데이터가 구체적인 개선 행동으로 이어집니다.
3. **개인정보 보호 (Privacy)**: 플레이어 개인정보를 보호하며 데이터를 수집합니다.
4. **실시간 모니터링 (Real-time Monitoring)**: 중요 지표를 실시간으로 추적합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 KPI 정의 (KPI Definition)
- **핵심 KPI**: DAU, MAU, 리텐션, ARPU 등
- **게임 특화 KPI**: 레벨 완료율, 평균 플레이 시간 등

### 4.2 텔레메트리 (Telemetry)
- **이벤트 정의**: 수집할 이벤트와 파라미터
- **수집 빈도**: 실시간, 배치 등

### 4.3 분석 & 대시보드 (Analytics & Dashboard)
- **분석 계획**: 주요 분석 시나리오
- **대시보드**: 실시간 모니터링 화면

### 4.4 선택 및 심화 (Optional & Advanced)
- **분석 도구 선택**: Google Analytics for Games, Unity Analytics, GameAnalytics, Amplitude, Mixpanel, 비용 vs 기능 비교
- **데이터 파이프라인**: 실시간 스트림(Kafka, Kinesis) vs 배치 처리(Hadoop, Spark), 데이터 웨어하우스(BigQuery, Redshift) 구축
- **이상 탐지 자동화**: KPI 급변 시 자동 알림(Slack, Email), 머신러닝 기반 이상 패턴 탐지, 치팅 탐지 알고리즘
- **개인정보 보호 전략**: GDPR/CCPA 준수 체크리스트, 익명화(Anonymization) vs 가명화(Pseudonymization), 데이터 보관 기간 정책(30일/90일/1년)
- **A/B 테스트 프레임워크**: 플레이어 그룹 분할(50:50, 90:10), 통계적 유의성 검증(p-value < 0.05), 최소 샘플 크기 계산

## 5. 관련 개념 (Key Concepts)

### 5.1 핵심 지표
- **KPI (Key Performance Indicator)**: 핵심 성과 지표
- **DAU/MAU**: 일간/월간 활성 사용자
- **Retention**: 플레이어 유지율 (D1, D7, D30)
- **ARPU**: 사용자당 평균 수익 (Average Revenue Per User)

### 5.2 데이터 수집
- **Telemetry**: 게임 내 플레이어 행동 데이터 수집
- **Event Tracking**: 특정 이벤트 발생 추적 (레벨 완료, 아이템 구매 등)
- **Session**: 플레이어의 게임 플레이 세션 (시작-종료)

### 5.3 분석 & 도구
- **Funnel Analysis**: 플레이어 여정 단계별 이탈률 분석
- **Cohort Analysis**: 동일 시기 유입 플레이어 그룹 분석
- **A/B Testing**: 두 버전 비교 실험

### 5.4 데이터 해석 함정
- **Use Rate vs Win Rate**: 메타 건강도의 1차 지표는 사용률, 승률은 슬리퍼(저사용·고승률) 탐지용 보조 지표. '저사용률+고승률'은 건강한 스파이스 요소의 시그니처일 수 있음 (Clash Royale 밸런싱)
- **Sample Bias (표본 편향)**: 소수 헤비 유저나 특정 획득 경로가 집계 평균을 왜곡하는 현상. 평균을 읽기 전에 표본 출처 분해가 선행되어야 함
- **분석 위계**: 퍼널(이탈 지점만) < 히트맵(공간 맥락 포함) < 플레이 영상(전체 맥락). 정보량과 분석 비용이 함께 증가하므로 문제 심각도에 따라 단계 선택

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 KPI
- [ ] 모든 핵심 KPI가 정의되었는가?
- [ ] KPI가 측정 가능하고 명확한가?
- [ ] 목표 값이 설정되었는가?

### 6.2 텔레메트리
- [ ] 중요 이벤트가 모두 수집되는가?
- [ ] 개인정보 보호가 고려되었는가?
- [ ] 데이터 수집이 게임 성능에 영향을 미치지 않는가?

### 6.3 분석
- [ ] 분석 계획이 구체적인가?
- [ ] 대시보드가 실시간으로 업데이트되는가?
- [ ] 데이터가 의사결정에 활용되는가?
- [ ] 지표 급변 시 표본 출처(획득 경로·유저 편중·이벤트 타이밍)를 분해해 확인하는가?
- [ ] 자기보고 피드백(설문·인터뷰)을 텔레메트리와 교차검증하는가?
- [ ] 실패 입력(실패한 점프·오조작)이 공간 좌표와 함께 수집되는가?
- [ ] '메트릭으로 결정할 것'과 '비전으로 결정할 것'의 경계가 문서에 명시되었는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 핵심 KPI**
- **좋은 예 (Good)**: "DAU: 일간 활성 사용자 수, 목표 10,000명 / D1 리텐션: 40% 목표 / ARPU: 사용자당 평균 수익 $2, 모든 KPI가 측정 가능하고 목표 명확"
- **안티패턴 (Bad)**: "플레이어 만족도 같은 추상적 지표만 정의하여 측정 불가"

**Case 2: 텔레메트리 이벤트**
- **좋은 예 (Good)**: "레벨 시작/완료, 아이템 구매, 플레이어 사망, 각 이벤트마다 타임스탬프·플레이어ID·레벨ID 등 파라미터 포함"
- **안티패턴 (Bad)**: "과도한 데이터 수집으로 성능 저하 및 개인정보 침해"

**Case 3: 실시간 대시보드**
- **좋은 예 (Good)**: "DAU·리텐션·ARPU를 실시간 그래프로 표시, 이상 징후 시 알림, 팀 전체가 접근 가능"
- **안티패턴 (Bad)**: "데이터 수집만 하고 분석·활용하지 않음"

**Case 4: Path of Exile - 플레이어 행동 분석**
- **구현 사례**: Path of Exile은 매월 데이터 공개 리포트를 통해 플레이어 행동을 분석하고 게임 밸런싱에 반영
- **추적 메트릭**: 클래스별 선택율(Marauder 14%, Witch 18% 등), 스킬 사용률(Fireball 12% vs 5% 변화 추적), 리그 별 리텐션(1주차 65% → 4주차 35%), 경제 지표(아이템 드롭율, 가격 지수)
- **데이터 기반 결정**: D30 리텐션이 예상보다 5% 낮으면 즉시 몬스터 드롭율 10% 상향, 특정 스킬 사용률이 2% 이하면 밸런스 패치 예정
- **개선 사례**: 2023년 분석 결과 새로운 플레이어의 D7 리텐션이 32%로 낮아 튜토리얼 난이도 조정 후 40%로 상향

**Case 5: Genshin Impact - 실시간 모니터링과 즉각적 대응**
- **수집 데이터**: 캐릭터별 사용률(Genshin에서 인기 캐릭터는 월평균 65% 이상 선택), 콘텐츠별 완료율(심연 12층 완료율 45%), 우편 클레임 속도(신규 플레이어 평균 세션 내 5분 이내 클레임)
- **ARPU 추적**: 초기 7일 ARPU $3.50 → 30일 ARPU $12.80 → 90일 ARPU $25.40으로 추적하여 갓차(캐릭터 소환) 시스템 최적화
- **A/B 테스팅**: 스피로 드롭율 테스트(10% vs 15%)를 1만명씩 비교해 D30 리텐션 2% 차이 확인 후 최적값 선택
- **배포 후 분석**: 신 이벤트 "해상도 탐험" 런칭 후 첫 주말에 참여율 72% 달성, 2주차 55%로 감소 데이터 기반 2주차 보상 1.5배 증가

**Case 6: Rainbow Six Siege - 밸런스 데이터 기반 오퍼레이터 조정**
- **상세한 연산자별 분석**: 각 오퍼레이터의 선택율, 승률, 처치율, 사망율을 플랫폼별(PC/콘솔)·랭크별로 추적
- **구체적 수치 예시**: Thermite 승률 52% → 방벽 시간 3초 감소 후 49%로 조정, IQ 선택율 8% → Commando 9mm 탄약 증가 후 12%로 개선
- **이상 탐지**: 핫픽스 배포 후 초기 12시간 모니터링으로 크래시 발생 시 즉시 롤백(평균 응답 시간 45분)
- **플레이어 세그먼트 분석**: 골드 이상(상위 15%) vs 실버/브론즈(하위 50%)로 분리하여 다른 밸런스 기준 적용, 상위 랭크와 하위 랭크의 승률 격차 10% 이상 시 조정

**Case 7: Fortnite - 실시간 이벤트 기반 플레이어 참여 분석**
- **수집 메트릭**: DAU(일간 활성 사용자), 시간대별 플레이어 분포, 신규/복귀 플레이어 비율, 게임 모드별 선호도(배틀로얄 vs 크리에이티브)
- **구체적 수치**: 신 패스 시즌 런칭 시 첫 주 DAU 약 350만 명 → 2주차 290만 명(약 17% 감소), 체계적 로그인 트리거(일일 챌린지)로 3주차 복귀율 35% 달성
- **이벤트 기반 대응**: 대규모 라이브 이벤트(방송) 시 동시 접속자 수 실시간 추적(약 1400만 동시 접속 기록), 서버 용량 초과 감지 시 자동 대기열 조정
- **ARPU 추적**: 배틀패스 구매율 45%, 스킨 구매 추가도 50%(평균 플레이어당 월 $15-20 ARPU), 한정 판매 스킨 가용성에 따른 판매량 변동 추적
- **세그먼트별 분석**: 캐주얼 플레이어(월 5-10시간) vs 코어 플레이어(월 50시간 이상)로 구분하여 다른 보상 정책 적용, 캐주얼층 리텐션 D7 42% → 이벤트 개선 후 48%

**Case 8: Apex Legends - 캐릭터 메타 분석과 밸런스 패치**
- **캐릭터별 추적**: 35개 캐릭터의 선택율, 승률, 생존 시간, 처치 참여도를 매주 추적
- **구체적 수치 사례**: Wraith 선택율 18% → Tactical 쿨타임 증가 후 12%로 조정, Valkyrie 선택율 24% → 수직 이동성 제한 후 15%로 균형 맞춤
- **리그 별 세분화**: Pred(상위 0.2%)의 메타와 골드(상위 30%)의 메타를 별도 추적, 같은 캐릭터도 랭크별로 승률 차이 8-15% 발생하므로 다른 밸런스 기준 적용
- **패치 후 즉각 모니터링**: 핫픽스 후 첫 48시간 플레이 시간대별 데이터 수집으로 메타 변화 추적, 부작용 감지 시 2-3일 내 재조정
- **A/B 테스팅 사례**: 신 총기 DMG 140 vs 160 두 그룹 테스트(각 50만 플레이어), 140 DMG 그룹의 KD 레이트 0.98 vs 160 DMG 그룹 1.05 → 160 선택

**Case 9: Dota 2 - 영웅 픽률·밴율 기반 대규모 밸런스 패치**
- **광범위한 수집 범위**: 123개 영웅의 픽률, 밴율, 승률을 5분 간격으로 실시간 추적, 게임 시간대별(초반 0-10분 vs 후반 40분+) 성능 분리 분석
- **구체적 수치**: 디바인 랭크(상위 0.1%)에서 Lion 픽률 22% → 밤의 장막 쿨타임 20초 → 30초 증가 후 12%로 하향 조정
- **패치 영향도 분석**: 월간 대규모 패치마다 약 30-50개 영웅 조정, 각 수정 사항별로 영향도 예측 모델 활용(변경 크기별 예상 승률 변화율)
- **세계 지역별 메타**: 동아시아(중국/한국) vs 서방(EU/NA) vs 남미의 서로 다른 메타 추적, 같은 영웅이 지역별로 승률 5-20% 차이 발생하므로 글로벌 밸런스 전략 필요
- **라이브 스트림 인테그레이션**: 프로 경기 중 픽률·밴율 실시간 표시로 시청자 분석 강화, 프로 플레이와 일반 플레이의 메타 괴리도 추적

**Case 10: 표본 편향 탐지 (Slay the Spire)**
- **사건**: Madness 카드의 승리덱 등장률이 급등했으나, 원인은 카드 성능이 아니라 '3막 후반 이벤트에서 지급'(승리 직전 획득)이라는 표본 출처 문제였음. 초기 데이터는 중독적으로 플레이한 슈퍼 테스터 2명이 대부분을 생산해 집계 평균이 사실상 그 2명의 표집이었음
- **좋은 예 (Good)**: "지표 급등 시 획득 경로·유저 분포를 먼저 분해한 뒤 밸런스 판단"
- **안티패턴 (Bad)**: "등장률 수치만 보고 즉시 너프, 실제 원인(지급 타이밍)은 미해결로 잔존"

**Case 11: 실패 입력의 공간 시각화 (Uncharted 3)**
- **구현**: 점프를 눌렀는데 렛지를 잡지 못한 위치의 XYZ 좌표를 전부 기록해 게임 내 빨간 구체로 임포트하는 'Bad Jumps' 시스템, 구축에 수 시간 소요
- **효과**: '잡을 수 있어 보이지만 못 잡는' 아트 요소 아래 구체가 군집해 아티스트가 즉시 수정. 사망 수 테이블+엑셀 조건부 서식 히트맵도 같은 저비용 즉효 분석
- **안티패턴 (Bad)**: "실패 횟수만 집계해 어디서·왜 실패하는지 공간 맥락 부재"

**Case 12: 원인 귀속 타임라인 (League of Legends)**
- **사건**: Poppy 승률 스파이크가 리워크와 마스터리 변경 중 어느 쪽 탓인지 팀 내 논쟁 발생
- **해결**: 데이터를 시간순으로 되짚어 스파이크 시작이 마스터리 변경보다 앞선 것을 확인하고 둘 다 조정. 한쪽만 건드렸다면 다음 패치에도 미해결이었을 것
- **교훈**: 변경 사항이 겹칠 때는 지표 변화 시점과 배포 시점을 타임라인으로 대조해 원인을 귀속하라

## 8. 핵심 인사이트 (Key Insights)

### 8.1 지표 설계 원칙
- **측정 가능한 KPI**: 모든 중요 지표를 정량적으로 측정 가능하게 정의하고, 목표 값을 설정하세요. 게임 모드별, 플레이어 세그먼트별로 KPI를 분화시키세요.
- **실행 가능한 데이터**: 데이터 수집이 구체적인 개선 행동으로 이어지도록 분석 계획을 수립하세요. 2-3일 내 패치 적용 가능한 인사이트를 우선순위로 추출하세요.
- **개인정보 보호**: GDPR, CCPA 등 개인정보 보호 법률을 준수하며 데이터를 수집하세요. 익명화 처리 후 분석하여 플레이어 신뢰를 구축하세요.
- **실시간 모니터링**: 중요 지표를 실시간으로 추적하여 문제를 빠르게 발견하고 대응하세요(45분 이내 이상 상황 감지 및 대응).
- **세분화된 분석**: 플레이어 계층(캐주얼 vs 코어), 지역, 플랫폼, 랭크 구간별로 별도의 메타 분석을 수행하고 차등 정책을 적용하세요.
- **승률보다 사용률 (Clash Royale)**: 메타 건강도를 직접 보여주는 것은 사용률이고 승률은 슬리퍼 탐지용입니다. 목표 사용률은 요소의 역할(주력/별미)별로 다르게 설정하세요.
- **킬 지표와 목표 기여 지표를 분리하라 (Battlefield 3, 13,000명 연구)**: 연령이 높을수록 킬·점수는 줄지만 승률 하락은 훨씬 작았습니다(오브젝티브 플레이 경향). 킬 중심 단일 지표는 플레이 스타일 차이를 실력 차이로 오독하게 만듭니다.

### 8.2 데이터 해석 함정
- **데이터는 거짓말한다 — 표본 출처를 캐라 (Slay the Spire)**: 집계 수치 이전에 '누가, 어떤 경로로' 그 데이터를 만들었는지 분해하세요. 소수 헤비 유저 편중과 획득 타이밍 편향이 대표적 왜곡원입니다.
- **자기보고는 텔레메트리로 교차검증 (God of War)**: 난이도가 완벽하다며 전략 분석까지 하던 열성 테스터가 실제로는 easy 난이도 플레이어였습니다. 영상+텔레메트리+설정값으로 플레이어 프로필을 만들어 피드백의 맥락을 확정하세요.
- **메트릭은 뒤만 본다 (Extra Credits)**: 텔레메트리는 이미 존재하는 것에 대한 반응만 측정하므로 혁신을 만들 수 없습니다. KPI 문서에 '메트릭으로 결정할 영역'과 '비전으로 결정할 영역'의 경계를 명시하세요.
- **메트릭은 모델의 재료지 직접 min/max할 대상이 아니다 (Albion Online)**: 지표만 보고 최적화하면 국소 최댓값에 갇힙니다. "무엇을 알고 싶은가, 어떤 데이터를 모을 것인가, 긍정/부정 결과는 어떤 모양인가"를 수집 전에 명문화하세요.

### 8.3 수집 전략
- **전량 수집은 수동적 플레이테스트다 (Slay the Spire)**: 런 종료마다 전 데이터를 수집하면 피드백을 남기지 않는 조용한 유저의 플레이도 전량 데이터화됩니다. 2인 팀이 수백 장 카드×유물 조합 밸런스를 감당한 기반. 스트리머는 최고의 정성 데이터 소스입니다(생각을 소리 내고 개발자 눈치를 안 봄).
- **리텐션은 계단식 폭포 (Design Home)**: D2를 35%→40%로 올리면 D3~7이 따라 오릅니다. 개선 A/B는 신규 유저 95:5 분할로 리스크를 누르고 시작하세요. 온사이트 테스트는 첫 45분만 검증하므로 7일차 문제는 다이어리 스터디+텔레메트리 레이어링으로만 잡힙니다.
- **비접속 관여 지표도 실측하라 (Path of Exile)**: 리그 말기 접속자가 피크의 20%까지 빠져도 웹 트래픽은 50% 밑으로 떨어지지 않았고, 이 수치가 3년 연속 다음 리그 서버 수요를 적중시켰습니다. '안 하는 동안에도 패치노트를 읽는' 관여 상태를 지표화하세요.
- **로그는 상대값·의미 단위로 저장하라 (Forza / Killer Instinct)**: Forza는 트랙을 12가지 턴 타입으로 분해해 세그먼트 단위로 기록해 처음 보는 트랙에도 일반화했고, Killer Instinct는 체력·거리를 상대값으로 기록해 같은 패턴을 다른 상황에 재사용했습니다. 로그 스키마 설계가 재사용성을 결정합니다.
- **거래 네트워크 시각화로 조작범을 잡는다 (NetEase)**: 원=플레이어(크기=거래액), 선=거래로 전체 거래망을 그리면 한 명이 수십 계정과 고빈도 양방향 거래를 반복하는 고립된 '꽃잎' 클러스터가 드러납니다. '중앙은행 대시보드'(중앙 통화 스톡, 좌측 발행 소스, 우측 회수 싱크)로 당일 통화량 안정 여부를 즉단하세요.
- **커뮤니티 불만은 3단 프로세스로 처리하라**: (1) 데이터로 실재 확인 → (2) 원인 분해 → (3) 유저가 제안한 해결책이 아닌 근본 문제에 대응. Valve는 2006년 Half-Life 2: Episode One에서 이미 사망 위치 히트맵으로 난이도 스파이크를 찾아 수정했습니다 — 이 프로세스를 운영 문서에 명문화하세요.
