# 코어 경험 & 코어 루프 작성 가이드

## 1. 목적 (Purpose)
게임의 핵심 경험과 반복되는 게임플레이 루프를 명확히 정의하여 플레이어가 지속적으로 참여하도록 유도하기 위한 가이드입니다.

## 2. 범위 (Scope)
- **포함**: 코어 경험 정의, 코어 루프 설계, 피드백 메커니즘, 플레이어 여정, 반복성과 변화의 균형
- **제외**: 구체적인 메커닉 구현, 기술적 세부사항, UI 디자인

## 3. 설계 목표 (Design Goals)

1. **명확한 코어 경험 (Clear Core Experience)**: 플레이어가 게임에서 반복적으로 느끼는 핵심 감정과 경험을 정의합니다.
2. **중독성 있는 루프 (Addictive Loop)**: 행동 → 피드백 → 보상 → 다음 행동의 순환이 자연스럽고 만족스럽습니다.
3. **지속적 동기부여 (Continuous Motivation)**: 단기·중기·장기 목표의 균형으로 지속적인 참여를 유도합니다.
4. **변화와 깊이 (Variety & Depth)**: 반복 플레이에도 지루하지 않도록 변화와 깊이를 제공합니다.

## 4. GDD 문서 구조 (Structure)

### 4.1 코어 경험 (Core Experience)
- **핵심 감정**: 플레이어가 느끼는 주요 감정 (긴장, 성취, 탐험 등)
- **플레이어 여정**: 전체 게임 경험의 흐름

### 4.2 코어 루프 (Core Loop)
- **기본 루프**: 행동 → 피드백 → 보상 → 진행
- **루프 변형**: 상황별·레벨별 루프 변화

### 4.3 피드백 & 보상 (Feedback & Rewards)
- **즉각적 피드백**: 모든 행동에 대한 즉각적 응답
- **보상 구조**: 즉각적·단기·장기 보상 분배

## 5. 관련 개념 (Key Concepts)

- **Core Experience**: 게임의 핵심적인 플레이어 경험
- **Core Loop**: 반복되는 기본 게임플레이 사이클
- **Feedback Loop**: 행동 → 피드백 → 학습의 순환
- **Progression**: 플레이어 성장과 발전
- **Flow State**: 도전과 기술의 균형으로 달성하는 몰입 상태
- **Empowerment**: 플레이어의 선택과 행동이 게임 세계에 미치는 영향력
- **Accessibility**: 모든 플레이어가 장벽 없이 게임 경험에 참여할 수 있는 능력

## 6. 품질 체크리스트 (Quality Checklist)

### 6.1 코어 경험
- [ ] 핵심 경험이 명확하게 정의되었는가?
- [ ] 플레이어 여정이 감정적으로 만족스러운가?
- [ ] 코어 경험이 게임 비전과 일치하는가?

### 6.2 코어 루프
- [ ] 루프의 각 단계가 명확한가?
- [ ] 루프가 자연스럽고 중독성 있는가?
- [ ] 루프 반복이 지루하지 않도록 변화가 제공되는가?

### 6.3 피드백 & 보상
- [ ] 모든 행동에 즉각적인 피드백이 제공되는가?
- [ ] 즉각적·단기·장기 보상의 균형이 적절한가?
- [ ] 보상이 플레이어 노력에 비례하는가?

## 7. 예시 및 안티패턴 (Examples & Anti-Patterns)

**Case 1: 명확한 코어 루프**
- **좋은 예 (Good)**: "적 처치 → 경험치·아이템 획득 → 레벨업·장비 강화 → 더 강한 적 도전, 각 단계마다 명확한 피드백"
- **안티패턴 (Bad)**: "행동과 보상의 연결이 불명확하여 플레이어 혼란"

**Case 2: 점진적 보상 (Dark Souls 2)**
- **좋은 예 (Good)**: "즉각적(소울 획득) + 단기(장비 해금) + 장기(지역 개방)의 계층적 보상 구조"
- **안티패턴 (Bad)**: "모든 보상이 즉각적이어서 장기 동기 부족"

**Case 3: 루프 최적화 (Roguelike)**
- **좋은 예 (Good)**: "기본 루프 유지 + 무작위 요소로 매 플레이 변화 + 메타 프로그레션으로 장기 성장"
- **안티패턴 (Bad)**: "완전히 동일한 루프 반복으로 지루함 유발"

## 8. 핵심 인사이트 (Key Insights)

- **명확한 루프**: 플레이어가 행동·피드백·보상의 순환을 명확히 이해하도록 설계하세요.
- **즉각적 피드백**: 모든 행동에 0.1초 이내의 피드백을 제공하여 플레이어 학습을 돕고 만족도를 높이세요.
- **계층적 보상**: 즉각적·단기·장기 보상의 균형을 유지하고, 경험 자체가 보상이 되도록 하세요.
- **루프 진화**: 기본 루프를 유지하면서도 변화와 깊이를 제공하여 반복성을 극복하세요.
